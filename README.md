# 32B 税务大模型 · think 去检索腔强化学习（32b-RL）

把公司 **V1**（Qwen2.5-32B 微调的税务模型）的推理过程 `<think>`，从"念手册、查 RAG 的检索腔"
（"参考问答对1 / 资料显示 / 检索结果表明"…）改造成"像人一步步推"，**同时锁死 `<answer>` 不漂离 V1 自己认可的答案**。
约束是：要强化的 V1 头上没有更强的老师（用更强模型蒸馏 = 公司已经训崩的 V2 老路），所以方法必须是**纯模型自我改进的 RL**。

> 这是一个**已完成**的项目，本仓库是重构后的最终交付：干净的最终链路代码 + 一份权威实验报告 + 完整历史归档。

---

## 一句话结论

四阶段自我改进 **冷启动 SFT → RFT → DPO → 在线 GRPO**，最终在 500 题冻结验收集上：

| 阶段 | think 干净分（Kimi 0–10）| 规则去检索腔通过率 | 答案在池率 |
|---|---:|---:|---:|
| V1 基线 | 3.140 | 2.6% | 93.8% |
| 冷启动 SFT | 4.408 | 45.4% | 83.0% |
| RFT | 4.489 | 45.6% | 85.0% |
| **DPO（最优）** | **4.643** | 49.0% | 84.6% |
| 在线 GRPO | 4.601 | 50.0% | 83.8% |

- **DPO 相对 RFT think 干净分 +0.155，刚越过"真涨门"（≈3×标准误≈0.15）= `DPO_TRUE_GAIN`** —— DPO/GRPO 这条线
  第一次量出"强化学习把 think 去检索腔"是真能学到的（不是裁判噪声）。
- 这之前在另一套老验收集上跑过**四轮 RL，净提升≈0**。但每一次"没涨"拆开都不是"RL 不行"，而是
  **测量与选样没对准目标**（de-RAG 把 grounding 一起去掉 / 奖励把答案和 think 串台 / 裁判噪声盖过信号 /
  优化了一个 60% 假阳的指标）。把"测量仪器和学习信号对准目标"做对之后，RL 就涨了。

完整来龙去脉（每一步怎么做、为什么、踩了哪些坑）见 **[`32B强化学习_实验报告.md`](32B强化学习_实验报告.md)**。

## 验收用的三个量（"三件套"，一个工具只判一件事）

| 判什么 | 谁判 | 单/多次 |
|---|---|---|
| think 有没有"换词复述照抄" | **Kimi**（DashScope，0–10 干净分） | k=3 取平均 |
| think 有没有表面检索腔词 | 规则 `detect_rag_style`（确定性、零噪声） | 单次 |
| 答案有没有漂离 V1 认可范围 | 规则 `answer_in_v1_pool`（确定性、零噪声） | 单次 |

> 为什么先标定 Kimi：唯一靠大模型判的"换词复述"很抖，开训前先用 78 条标定集量清它的分辨率
> （judgecal），得出"逐条选样只能用'基本干净 vs 重度照抄'大间距对子""整体评测靠大 N 摊薄、真涨门≈0.15"
> 两条铁律——这直接决定了 DPO 对子的造法和真涨判据。
