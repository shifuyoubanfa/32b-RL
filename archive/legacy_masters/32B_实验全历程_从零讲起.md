# 32B 税务模型强化学习 · 实验全历程（从零讲起）

> 按时间顺序、第一人称，把整个项目讲一遍，给完全没接触过的人看。
> **每个阶段统一用五段式**：① 我在做什么 → ② 数据流（每个输入/输出都说清：哪个模型产生、哪个数据集多少条、谁用什么阈值从 N 筛到 M、输出存哪）→ ③ Kimi 在这步的角色（+ 首次出现的提示词全文）→ ④ 规则/奖励的角色（+ 完整数学式）→ ⑤ 结果 / 错在哪（+ 这结果怎么逼出下一阶段）。

## 开篇：两个主角 + 一套数据底盘

**两个打分的角色**（全程就这两个东西在给模型输出打分）：
- **Kimi**（外部大模型，公网 DashScope，model=kimi/kimi-k2.6）：干四种活——(1) 评测裁判；(2) 改写工（把脏思考改干净）；(3) 成对比较；(4) 训练门禁。**通病：打分有噪声、不稳定。**
- **规则**（本地奖励函数，纯正则+字符串相似度，不调大模型）：干两种活——(1) 当强化学习 reward；(2) 评测时数痕迹。**通病：只认字面，看不见"换了词的念手册"。**

**一套数据底盘**（来自 `config.py`，全程不变）：
- **总数据 2239 条**（`00_data_teacher_outputs`，每条 = 用户问题 + 参考问答对 + V1 的 think/answer）。
- 按 9:1 切成 **训练集 2015 条**（`SFT_TRAIN`）+ **验收集 224 条**（`SFT_EVAL`，**全程冻结，每个模型都在它上面考**）。（config.py 注释里写 2014 是笔误，实际落盘 2015 条。）
- **224 验收集是"谁评的"，各阶段不一样**（后面每步也会标）：**阶段0–4** 用同一把 **Kimi 拟人度裁判（单次判分）**；**阶段5** 换成 **Kimi 去 RAG 裁判（每条打 3 次取平均）**；**阶段6** 用**纯规则的 frozen 痕迹计数器**（不调 Kimi）；**阶段8** 思考用规则/Kimi、答案用规则。**谁评的不同，数字就不能跨阶段直接比。**
- 两套系统提示：原始 V1 用 `SYSTEM_PROMPT`（"基于参考问答对搜索回答"，RAG 腔来源）；所有训练后的模型用 `COLDSTART_SYSTEM_PROMPT`（"像资深税务老师自然推导，别复述参考问答对"）。

**全程我盯三个数**：像不像人（念手册腔）、答案准不准（跑没跑偏）、有没有照抄。
**全书主线**：每一次"强化学习没涨"，拆开都是 Kimi 打分 或 本地奖励函数 这两个打分标准没对准目标，不是强化学习本身不行。

## 阶段 0 · 我先给原始模型做体检

**① 我在做什么**：原始 V1 不改，在 224 冻结验收题上考一遍，看清起点。

**② 数据流**：
- 输入模型：**原始 V1**（系统提示 = `SYSTEM_PROMPT`，RAG 腔）。
- 输入数据：**224 验收题**（`SFT_EVAL`，冻结）。
- 采样：贪心（温度 0），每题 1 份 → **224 条推理**。
- 谁筛：不筛。
- 输出：224 条推理 + Kimi 三项分（`10_base_report.md`）。

**③ Kimi 的角色**：**评测裁判**（第一次出现）。提示词全文：
```
[系统] 你是严格、客观的中文税务问答评审专家。只输出 JSON，不要任何额外文字。
[模板] 请评审一个税务模型的「推理过程(think)」和「答案(answer)」。
【问题】{query}【参考资料】{reference}【标准答案】{gold}【待评审 think】{think}【待评审 answer】{answer}
评审四件事，严格按 JSON 输出：
1. accuracy：answer 与标准答案是否一致，取 "correct/partial/incorrect"，accuracy_score 0~1。
2. humanness：think 像不像「人在端到端推理」、越没 RAG 痕迹越高，0~1。
   锚点：0.9~1.0 像人推导；0.0~0.2 出现"参考问答对1/2/3""根据检索结果"或大段照搬政策原文。
   即使没这些词，若 think 是"从资料向答案归纳"而非"从问题推导"，也偏低。
3. grounded：think 的依据/口径/数字/结论是否忠于参考资料、不矛盾，0~1。
4. rag_traces：痕迹类型 ["explicit_ref","verbatim_copy","ref_enumeration","policy_source"]。
只输出 JSON：{"accuracy":"...","accuracy_score":0.x,"humanness":0.x,"grounded":0.x,"rag_traces":[...],"comment":"..."}
```

**④ 规则的角色**：还没上场。

**⑤ 结果 / 错在哪**：像人 **0.309**、守依据 0.910、准确率 0.904（答对 177/224）；显式引用 222 次、逐字照抄 174 次。把题按答对/答错分组，像人都是 0.31——**它的准是抄出来的**。**埋雷：去掉抄，准会掉。**
**还有一个一开始没当回事、后来反复咬我的雷：这条基线、以及后面每一个对比数，都是 Kimi 单次打分量出来的**——没有"同一条打几次取平均"，而 Kimi 单次拟人度打分的标准差约 0.117。**也就是说，我用来判断"涨没涨"的那个对比基准，本身就在抖。** 后面好几轮"读不出涨没涨"，根子之一就是连基线都不可靠。

## 阶段 1 · 我用 Kimi 把"念手册"改成"人话"，做冷启动

**① 我在做什么**：教模型别照抄。我先用 Kimi 把训练集每道题的机器腔思考改写成人话，再拿改写数据做监督微调。

**② 数据流（三步）**：
- **step06 改写**：输入 = **原始 V1 产的 2015 训练题**（每题 原始机器腔 think + 参考 + 答案）→ **Kimi 改写工**（温度 0.3）逐条改 think，**不筛全改** → 输出 `30_seeds_rewritten` **2015 条**（含规则自检的 facts_ok/trace_hits/copy_ratio）。
- **step07 打分**：输入 = 上面 2015 条 → **Kimi 裁判**（温度 0）给每条打 humanness + 守依据，**不筛** → `30_seeds_scored` 2015 条。
- **step08 建冷启动集**：输入 = 2015 条 → **规则五道闸**筛 → 输出冷启动训练集。
  - 五道闸（`seed_is_chosen`）：`facts_ok（没引入新数字）∧ trace_hits==0（无痕迹词）∧ kimi_humanness≥0.60 ∧ grounded≥0.70 ∧ copy_ratio≤0.50`。
  - 从 **2015 条筛到 1498 条干净种子** → `40_coldstart_train` **1348 条** + 自然腔早停集 `40_coldstart_eval` **150 条**（1348+150=1498）→ 监督微调出**冷启动模型**。

**③ Kimi 的角色**：**改写工**（第一次出现）+ 打分。改写提示词全文：
```
[系统] 你是中文税务写作专家。把'查资料式'的机器推理，改写成'资深税务老师从问题出发自然讲解'的口吻——
       但依据、口径、数字、结论必须完全来自给定参考资料，绝不能凭通用知识发挥或与参考矛盾。
[模板] 请把【原始推理】改写得更像一个人在自然思考，严格满足：
1.【去检索腔】删掉"根据参考问答对/检索结果/参考资料/问题1的回答"之类字眼，改成从用户问题出发一步步推导。
2.【必须扣依据·最重要】依据、政策口径、数字、结论必须来自参考资料，把关键事实自然融进推导，绝不与参考矛盾。
3.【事实零改动】所有数字、税率、金额、日期、政策结论一字不改、不新增臆造、不删关键点。
4. 只输出改写后的推理，不要答案、不要解释。
【参考资料】{reference}【用户问题】{query}【原始推理】{think}
```

**④ 规则的角色**：**改写质检员 + 选种子**——step06 自检 `facts_ok`（防臆造新数字），step08 五道闸筛种子。**这步没有 reward（是监督微调）。**

**⑤ 结果 / 错在哪**：像人 0.309 → **0.680**；准确率 0.904 → **0.803**。**Kimi 改写时把"扣依据"和"照抄痕迹"一起删了**（指令既要去检索腔又要扣依据，它更听前半句），一批靠扣参考答对的题改偏了 = **去检索腔丢守依据**。

**这里要补一句当时没说破、其实很关键的话——"五道闸到底有没有筛出好数据"，我们从来没真正证明过。** 拆开看：① "像人 0.309→0.680"是 Kimi 单次打分量的、带噪声，就算姑且认它在 224 条平均下真涨了；② "准确率掉了 0.10"这件事说明，**模型并没有把"既去痕、又不丢依据"的高质量数据吸收进去**——它学到的更像是"少写几个痕迹词"，而把依据连带删了。**humanness 涨 + accuracy 掉，恰恰说明五道闸放进来的种子里混着"去痕但伤依据"的坏料**，可这道闸从没拿独立标准验证过它选的对不对。**结论：冷启动这步"涨了"是不可靠的，"模型真吃进好数据"则没有证据。**

## 阶段 2 · 我做拒绝采样微调（RFT），第一次祭出"本地奖励"

**① 我在做什么**：把冷启动掉的准确率捞回来——让模型对每题答 8 遍，用本地奖励打分，只留高分的再训。

**② 数据流（两步）**：
- **step10 采样**：输入 = **冷启动模型**在 **2015 训练题**上，每题 **K=8 遍**（温度 0.9、top_p 0.95，系统提示 COLDSTART）→ `50_cs_rollout` **2015 条**（每行 8 候选）。
- **step11 选样**：输入 = 2015×8 = 16120 个候选 → **本地奖励函数分两层挑** → `50_cs_rft_train` **1863 条**（约每题留 1 条）→ 微调出 **RFT 模型**（这一步在 224 上评像人 **0.704**，见报告 50）。
  > 一个口径说明：后面把这个 RFT LoRA 和 V1 合并成一个完整模型、当作**全程固定的对比基准**，那个合并版重新评是**像人 0.697**（报告 80）。两个数是同一个模型的两次评测（adapter 版 0.704 / 合并版 0.697），**后文一律拿合并版的 0.697 当"全程最好的底子、对比基准"**。
  - **第一层（先过一道及格线）**：一个候选要同时满足"格式正常"（有 `<think>`/`<answer>` 两段、think 长度 40–2000 字、不是重复废话）**和**"答案没明显跑偏"（答案漂移分 R_acc ≥ 0.6）才留下。
  - **第二层（剩下的里挑最好）**：幸存候选按下面 ④ 那个总奖励排序，每题只留分最高的 1 个。

**③ Kimi 的角色**：只当**评测裁判**复测（复用阶段 0），不参与训练打分。

**④ 规则的角色**：第一次当 **reward**。完整数学式（`reward.py`）：
```
copy   = max( 最长公共子串/think长度 , think与参考的5-gram重叠率 )          # 照抄率
hits   = think 命中 14 条 RAG 痕迹正则的总次数（"参考问答对"/"根据检索"/"〔数字〕号"...）
R_human = 1 / ( 1 + 0.34·hits + 1.00·copy )                                # 拟人项（PMI 关，就这一项）
R_acc   = 0.5·SequenceMatcher相似度(V1的答案, 模型答案) + 0.5·关键事实召回   # 答案漂移项；V1的答案只给reward打分，不进模型输入

总奖励：  格式不合法 → -1.0 ；  R_acc<0.30 → 0.1·R_acc ；  否则 → R_acc·( 0.5 + 0.5·R_human )
```
公式里**两个 0.5 是权重超参**（代码里叫 REWARD_W_ACC、REWARD_W_HUMAN，都设成 0.5，沿用 14B 微调时调出来的值），不是写死的魔法数。
**"关键事实"** 指答案里这几类东西：百分比（"13%"）、金额（"10 万元"）、日期（"2024 年 1 月"）、期限（"30 天"）、结论极性词（"免征/应缴"）。**"关键事实召回"** = 模型答案命中了 V1 答案这些关键事实里的几个，占 V1 答案关键事实总数的比例。举例：V1 答案有"13%、免征、10 万"三个关键点，模型答案命中两个 = 召回 0.67。

我还做了个验证测试：拿"已知高质量的 Kimi 改写"和"已知机器腔的原始输出"两批标准样本，看本地奖励函数能不能把它俩分开。这里用的指标叫 **AUC（把好样本排在坏样本前面的能力，1.0 满分、0.5 等于瞎猜）**。

**核心结论（单独说）：本地奖励函数在这两批样本上的 AUC=0.990，几乎完美——但这是道"送分题"**：机器腔有"参考问答对1"这种明晃晃的字面标志，数它字面词的规则一抓一个准。**等后面遇到"换了词、表面已经挺干净的结构性念手册"这种真考题，同一个本地奖励函数就崩**：它给的分和 Kimi 给的分相关系数（Spearman）只剩 **0.077**（几乎不相关，见阶段4），当门禁用时排序能力 AUC 只剩 **0.69**（见阶段6）。

**⑤ 结果 / 错在哪**：像人 0.704、准确率 0.813（只回来一点）。**埋了第二个雷，但它的"病"和我一开始讲的不一样，这里纠正**：`R_acc` 有一项 `SequenceMatcher(V1的答案, 模型答案)`，RL 会把模型答案不断推向 V1 答案（模型生成时看不到 V1 答案，gold 只在 reward 里打分，是 RL 间接把它推过去的）。**我之前把这条叫"照抄泵、是病根"——这个定性不对。** 把答案推向 V1 本来就是我们要的（目标就是"think 去痕 + 保留 V1 答案"，V1 贴参考、我们照着贴也认）。真正的设计硬伤是另外两条：
- ① **R_acc 用"字面相似度 + 事实召回"给的是连续的部分分**：一个锚定事实答错了，也只是少召回一点、照样能拿不低的分。**正确做法应是"锚定事实错 = 直接判 0（硬否决）"**，而不是给个相似度分让错答案蒙混过关。
- ② **答案这条信号被乘进了和 think 拟人度同一个奖励式里**（`R_acc·(0.5+0.5·R_human)`）：等于让"答案像不像 V1"和"think 干不干净"在一个分数里互相稀释——**该把答案做成一道硬门（事实对=放行、错=0），把优化信号全留给 think。**

这两条到下一阶段引爆。

## 阶段 3 · 第一轮强化学习（DPO + GRPO）——链路跑通，但奖励信号设计有硬伤（答/思串台、错事实给部分分）

**① 我在做什么**：用 DPO/GRPO 把基座的照抄磨掉。跑了原始链 + 修正参考模型版（corrected-v1）多个变体。

**② 数据流**：
- **DPO 对子（step12）**：输入 = **RFT 基座**在 2015 题各采 8 遍 = 16120 个候选 → 用阶段2 那个本地奖励函数给每个候选打总奖励分 → **每题挑一对**：分最高的当"好样本(chosen)"、分最低的当"坏样本(rejected)"，且两者分差 ≥ 0.05（太接近就不成对）→ 得 `60_dpo_pairs` **1516 对**（候选先过的"及格线"同阶段2 step11 第一层）。
- **GRPO 数据（step13）**：输入 = 训练集 2015 题，原样打包、不筛 → `70_grpo_data` 2015 条；GRPO 训练时再对每题现采 8 个、用同一个本地奖励函数当场打分。
- corrected-v1：把 DPO/GRPO 的"参照基准模型"换成冻结的 RFT 基座，重训 DPO + 两路 GRPO（一路从 RFT 起、一路从 DPO 起）。
- 输出：四个模型，全在 **224 冻结验收集**上、用阶段0 那把 Kimi 拟人度裁判（单次判分）评（汇总在 `84_summary`）。

> **一个常被问到的实现细节：DPO 训练更新梯度时，是整段都更新，还是只更新 think？**
> 当时喂给 DPO 的"好样本/坏样本"都是**完整的 `<think>…</think><answer>…</answer>`**，训练框架（ms-swift 的 DPO）也是**在整段输出上算损失**——所以 **think 和 answer 的每个字都进了梯度，我们没有把 answer 屏蔽掉。**
> 你设想的"反向传播时把 answer 完全忽略、只更新 think"是**可行、而且更对的另一种做法**：因为这一阶段一对里的好/坏样本**答案文本不一样**（没锁定），answer 的差异也跟着进了梯度——等于在改 think 之外、又顺带让模型去学"哪个答案更像 V1"，正好和"只想改思考、别动答案"拧着。后来 v5 方案提的 **"answer-lock"（一对里好/坏样本共用同一段答案、只让 think 不同）就是你这个思路**——但 derag_v4 当时没做。

**③ Kimi 的角色**：只当评测裁判。

**④ 规则的角色**：还是阶段 2 那个 `reward.py`——**既选 DPO 对子，又当 GRPO 在线 reward**。

**⑤ 结果 / 错在哪**：没有一个变体超过 RFT 基座 0.697，像人全负；唯一准确率涨的（0.831），逐字照抄从 60 涨到 63。
**关键问题：DPO 奖励里算了 answer 吗？——算了。但"病"在哪，我得纠正一下之前的说法。** 选 chosen 的综合奖励含 `R_acc = 0.5·SequenceMatcher(V1的答案, 模型答案) + 0.5·事实召回`，8 个候选里答案最贴 V1 的那个 R_acc 最高、被选成 chosen，DPO 把模型往它推 → 越训答案越贴 V1。**我之前把这叫"照抄泵、是病根"，定性不对**：答案贴向 V1 不是问题（保留 V1 答案就是目标）。真正的病根是——**这套 reward 把"答案像不像 V1"和"think 干不干净"乘在一个式子里、还对答案事实给连续部分分**，于是 DPO 的梯度被"挑答案更像 V1 的那条"主导，真正想改的"think 去痕"几乎没拿到独立优化信号。**这才是"准确率动了、像人却全负"的代码级根因。** 正确的拆法（也直接引出下一版设计）：先确认 SFT/RFT 没把答案能力做坏，再让**答案只当硬门（事实对就放行、错就 0），DPO/GRPO 的好坏对只比 think**，别让答案信号掺进来。

> **还有一个更底层、当时完全没意识到的隐患**：阶段2 那点"RFT 像人涨"，很可能就落在 Kimi 单次打分的噪声里——换句话说，**进这一轮 DPO/GRPO 之前，我们从没真正证明过"存在一个可学的强化学习信号"**。拿一个信号都没证实的基座去 DPO/GRPO，"没涨"几乎是注定的。这条直到阶段8 才被正面回答（那时才用探针证明"好样本确实存在"）。

## 阶段 4 · corrected-v2 · 我改用 Kimi 重选对子，发现"奖励和裁判根本不是一回事"

**① 我在做什么**：阶段 3 我怀疑"本地奖励函数在奖励照抄"。这一阶段分两幕——**先诊断**当时那个本地奖励函数准不准、它挑的 DPO 好坏对子可不可信；**再换 Kimi 从头重新打分、重选对、训个小实验**看能不能涨。

**② 数据流（分两幕：先诊断，再重做）**

**第一幕 · 诊断：当时的奖励函数、和它挑的对子，准不准？**
- **step90 查奖励函数准不准**：阶段 3 那四个模型，之前各自在 224 验收题上被 Kimi 打过分、留了记录（一共 4×224 = 896 条模型输出 + Kimi 给每条的拟人度）。我现在**用当时那个本地奖励函数，把这 896 条输出重新打一遍它的"像人分"（R_human）**，然后**把"奖励函数给的像人分"和"Kimi 当初给的拟人度"两两配对、算相关系数**，结果 **Spearman = 0.077（几乎是 0）**。
  - **因为这两套打分几乎不相关，所以判定：本地奖励函数衡量的"像人"，和 Kimi 衡量的"像人"根本不是一回事。** 这里要把话说严谨：这**只**证明"奖励和裁判量的不是同一个东西"，**并不能证明 Kimi 那套就对**——Kimi 对单条 think 打分本身也不可靠（同一条反复打标准差 0.117），规则更是只认字面、逻辑上本就有缺陷。**两套打分都没被验证成真值，能下的唯一结论是"它俩彼此不一致"，不是"规则错、Kimi 对"。** 所以这一阶段的标题该叫"奖励和裁判根本不是一回事"，而不是"谁对谁错"。
- **step91 查它挑的对子准不准**：既然奖励函数不准，它挑出来的旧 DPO 对子还能信吗？我从那批旧对子（`60_dpo_pairs`，就是奖励函数挑的）里抽 **150 对**，**让 Kimi 给每对的"好样本"和"坏样本"各打一次拟人度**，看"被奖励函数选成好样本的"是不是真比坏样本更像人。结果：**真的更像人的只占 34%，差距明显（拟人度差 ≥0.25）的只占 12.7%**。
  - **因为 Kimi 重新打分的结果和奖励函数的挑选结果对不上，所以判定：当时那批 DPO 对子也不准——大半是被一个不准的奖励函数瞎挑的。**

> **两幕之间**：奖励函数不准、它挑的对子也不准 → 那就**不再用这个本地奖励函数，改用 Kimi 从头重新打分、重新挑对子、训一个小实验**，看用 Kimi 重做能不能涨。

**第二幕 · 重做：换 Kimi 从头来，每一步都先自检**（编号跳过 92——那是"候选不够才重采样"的备用步，本次没触发）
- **step93 重新打分（先要有打过分的候选）**：重选对子得先有候选。为省钱我**不重新采样**，直接复用 **RFT 基座之前已生成好的候选池**（`60_dpo_rollout`，每题 8 个候选）。从里面随机抽 **120 题 × 8 = 960 个候选**，让 **Kimi 逐个打 拟人度/守依据/准确率**（温度 0、**单次判分**——注意这一阶段还没用"打 3 次取平均"，那是阶段5 才有的）→ 得到 960 个"带 Kimi 分"的候选（`93_rollout_scores`）。
- **step94 配对（用 Kimi 的分挑好坏对）**：对每道题的 8 个候选，用一套确定性规则挑出一对"好样本/坏样本"。这套规则要同时满足下面这几条——**好样本**：① Kimi 拟人度够高（先卡 ≥0.75）② 没有显式的念手册痕迹（"参考问答对"那类字眼）③ 守依据分 ≥0.8 ④ 答案和 V1 一致 ⑤ 照抄率 ≤0.30；**坏样本**：拟人度要比好样本低 ≥0.25（差距得够明显）；再加一条 ⑥ 一对里两条的长度别差太多（≤30%，免得模型把"短"当成"好"）。**第一次把好样本的拟人度门设到 ≥0.75，全 960 个候选只凑出 35 对**——不够训练（我定的下限是 80 对），**这一档直接判 NO-GO**（见 `94_报告`）。我把门**放宽到 ≥0.70**，凑出 **171 对训练 + 25 对留出**（见 `97_summary`）。
- **step95 复判（训之前先自检对子稳不稳）**：拿这 171 对去训之前，我先自检它们靠不靠谱——**隔一遍让 Kimi 重判一次这 196 对**，看"第一遍说 chosen 好"的，第二遍还认不认。结果**只有 59.2% 前后一致**（< 我要求的 70%）→ **NO-GO：近一半对子 Kimi 自己前后两遍都打架，说明它判分抖动太大、对子不稳。**
- **step95b 稳定过滤（既然整批不稳，只留最稳的）**：从 196 对里只挑**前后两遍都稳的那 63 对**，分成 **53 对训练 + 10 对留出**（`97_summary`）。
- **step96 mini DPO（用最稳的 53 对训一把看结果）**：拿这 53 对训 mini DPO，**在 224 冻结验收题上、用阶段0 那把 Kimi 拟人度裁判（单次判分）评全套指标**，对比 RFT 基座（括号内为基座值）：拟人度 **0.688（基座 0.697，降 0.009）**、守依据 0.853（0.858）、准确率 0.806（0.818）、答对率 61.2%（65.6%）、答对+部分对 90.6%（91.1%）。**五项全没涨、还都略降**（见 `96_报告`）。

**③ Kimi 的角色**：这一阶段 Kimi 顶上来当主力打分，干四种活——审旧对（91）、给候选打分（93）、第二遍复判（95，都复用阶段 0 那套评测打分），外加一种"成对比较"判法（让它直接判 A/B 哪个好，第一次出现）：
```
[系统] 你是严格、客观的中文税务问答偏好评审专家。只输出 JSON。
[模板] 比较同一问题下两段推理/答案，判哪个更符合目标。【问题】{query}【参考资料】{reference}【标准答案】{gold}【候选A】{a}【候选B】{b}
优先级：1. answer 不得更不准；2. think 须忠于参考、不臆测；3. 满足1/2下更像人从问题推导而非机械归纳；4. 不奖励更短，接近判 tie。
只输出 JSON：{"winner":"A|B|tie","confidence":0.x,"accuracy_risk":"A|B|none","reason":"..."}
```

**④ 本地奖励函数的角色**：这一阶段它是**被审查的对象**——step90 查出它和 Kimi 几乎不相关（0.077），就不再用它来挑对子了；只在 step94 配对时留作几条硬性检查（答案别跑偏、照抄率别太高）。

**⑤ 结论 / 错在哪**（这部分我去查了那 112 对的真实 Kimi 分数）：

**第一件事 · 当时的奖励函数、和它挑的对子，都不准。** step90 证明本地奖励函数和 Kimi 几乎零相关（0.077）；step91 证明它挑的旧对子里真更像人的只占 34%。补一句旧对子的来历：它是本地奖励函数"取分最高 + 取分最低"配出来的，**每题中间那 6 个候选 Kimi 当初从没打过分**——真正像人的好候选，可能就埋在那 6 个没被 Kimi 打过分的里，从没进过训练对。

**第二件事 · 换 Kimi 从头打分、重新挑了最好的，为什么还是没涨？** 这次 **Kimi 给每题全部 8 个候选都打了分**（step93，120 题 × 8 = 960 条），上面"中间 6 个没打分"的问题在这次不存在了。我把 112 对的真实分数拉出来看：
- **好样本（chosen，DPO 要往它推的那条）：Kimi 拟人度 均值 0.787、最低 0.75、最高 0.85**（这是 corrected-v2 这一轮、且**只统计"过了 ≥0.70 门"的好样本**，是被筛过的子集、所以均值偏高；别和阶段5 那个"全量未过滤 0.701"搞混，见阶段5 ⑤）；其中 **112 对里有 42 对正好卡在 0.85**——这是 Kimi 打分的老毛病："封顶在 0.85"，再好它也只给 0.85，分不出 0.85 和真正的 0.95。
- **坏样本（rejected）：拟人度 0.30–0.60、均值 0.388。**
- 所以**好样本确实比坏样本高一截（+0.4）、也比基座 0.697 高（+0.09）——不是"拿垃圾训垃圾"。**

**那为什么训完还是没涨？三个原因叠在一起，让这点 +0.09 的小提升根本读不出来：**
1. **好样本本身就不够好**：每题 8 个里最好的，Kimi 也只打到 ~0.79（最高卡 0.85 封顶）。DPO 顶多把模型往 0.79 这个水平推，提升上限就 ~+0.09；而且 0.85 是封顶，连真实质量到底多高都看不出。
2. **这一阶段 Kimi 是"单次打分"**（注意：**"同一条让 Kimi 打 3 次取平均"是下一阶段才引入的，这里还没有**）。单次打分波动大（同一条反复打，分数的标准差 0.117）→ 它选出的"最好候选"不可靠 → step95 隔一遍复判，只有 59.2% 还认同当初的选择。
3. **最后那个 224 验收，用的也是这同一把"单次打分的 Kimi 拟人度裁判"**（阶段0 那把）→ 评测本身就在抖。结果是：**这点 +0.09 的小提升，被评测的抖动盖住了，量出来是拟人度 −0.009——连到底涨了还是降了都分不清。**

**这一阶段还逼出一条以后一直用的开训前检查（顺便把 MDE 说清）。** 开训之前先算两个数、做对比：
- **预期提升（设计效应）= 这次训练大概能让指标涨多少。** 怎么估：拿"好样本比基座高多少"（这里 chosen 比基座高 ~0.09）乘上"模型大概能学进去几成"，估出这次约 **+0.006**。
- **MDE（最小可检出提升）= 在这套评测自己的抖动下，真提升至少要大到多少，才能在统计上和"没提升（0）"分得开。** 这套"224 题 + 单次 Kimi"的抖动，算出来 MDE 约 **0.022**。
- **打个比方**：好比一台最小只能称出 22 克变化的台秤，你想用它称"重了 6 克"——称不出来，6 克比它能分辨的最小变化还细。这里预期提升 0.006 比 MDE 0.022 还小，**所以哪怕模型真涨了，也一定淹在评测抖动里、读不出来。**
- **所以这是一个"开训前就注定读不出结果"的实验。** 这条后来定成铁律：**开卡之前先算"预期提升 > 评测能分辨的最小提升（MDE）"，不够就别烧卡。**

## 阶段 5 · corrected-v3.1 · 我换一套"新 Kimi 去 RAG 评分"，分清"评分能不能读出去痕" vs "模型能不能学会去痕"

> 说明：这一阶段的"新口径去痕度评分"**还是用 Kimi**——只是换了一套新提示词（专门判"有没有念手册痕迹"，不再判"像不像人"），而且**对同一条样本让 Kimi 独立打 3 次分、取平均**当结果，用来压住单次打分的波动。**不是规则。**（纯规则的痕迹计数器要到阶段6 才登场。）

**① 我在做什么**：阶段4 卡在"奖励函数不准"和"数据没好料"分不开。这一阶段我要**彻底排除"是不是 Kimi 评分本身不行"**：先换一套波动更小的 Kimi 去 RAG 评分，再用两个对照实验，把"**Kimi 评分能不能读出'去痕'**"和"**模型能不能学会去痕**"分开。

**② 数据流（三步：先给新评分做体检，再两个对照实验）**

**step110 · 先给新的 Kimi 评分做"波动体检"**
- 为什么：用一套评分之前，得先知道它**同一条样本反复打、分数会飘多少**——飘得比要测的信号还大，就没法用。
- **先说清 Kimi 每打一次分输出的是什么**：是一段 JSON，里面有 accuracy（答案对不对）、grounded（守不守依据）、和我们最关心的 **trace_free（去痕度，一个 0~1 的小数，越接近 1 越没念手册痕迹）**，外加一个 rag_traces（痕迹类型清单）。我们取其中的 trace_free 当"去痕度分"。
- 做法：拿 base（RFT 基座，编号 80）和 candidate（v2 mini_dpo，编号 96）**各 224 题的输出**，让新 Kimi 评分**对每一条独立打 3 次、取 3 个 trace_free 的平均**当这条的分（共 448 条 × 3 次）。
- 结果：**同一条样本反复打，分数的标准差从旧"拟人度"评分的 0.117 降到 0.056**——波动小了一半（落 `110_calibration`）。
- **0.056 到底算大还是算小？要看拿它干什么用**：
  - **判单独一条**（这一条够不够干净、能不能进训练）：0.056 是在 0~1 量程上的波动，意思是同一条这次打 0.72、下次可能打 0.66 或 0.78。而我们要分辨的"去痕"信号往往只有零点零几——**0.056 比这信号还大，所以靠单条打分下判断不可靠。**
  - **判一整批的平均**（这 224 条平均去痕度涨没涨）：取平均会把波动**大致按 √条数 缩小**——224 条平均后，波动 ≈ 0.056 ÷ √224 ≈ **0.0037**，**小到能稳稳读出 +0.038 这种整批变化**（见实验A），但仍读不出 +0.0005 那种（见实验B）。
  - **一句话：0.056 对"判单条"太大、对"判整批平均"够小**——这也是为什么 Kimi 适合给整个模型打总分、却不适合逐条挑样本（阶段6 还会撞上这点）。

**step112 · 实验A：先证"这套 Kimi 评分到底能不能读出'去痕'"**（编号 112 虽在后，但它是实验B的前提，先讲）
- 为什么：万一模型真去了痕、Kimi 评分却读不出，那后面一切白搭。所以先做个"阳性对照"——喂它一批**铁定去了痕**的样本，看它读不读得出。
- 怎么造"铁定去痕"的样本：拿 base 里 **60 条带痕迹**的输出，用一个**程序函数 `trace_surgery`**（不是 Kimi、不是人工，就是一串正则）把"检索腔帽子"和图片/链接**机械剪掉**。它删的就是下面这 11 类，**删完不改写、不动任何事实数字**：
  ```
  ① 根据(参考资料/上述资料/资料/检索结果/现有的回答)   ② 从现有的回答来看
  ③ 现有的回答(显示/提到/中)   ④ 参考(资料/文件/问答对/内容)(显示/表明/指出/中/里)
  ⑤ 资料(显示/表明/指出/中/里)   ⑥ 检索(结果/到的内容)(显示/表明/指出)
  ⑦ 这里提供了   ⑧ 图片链接   ⑨ 文件链接   ⑩ <img...> 标签   ⑪ https?:// 开头的链接
  ```
  举个真实例子，**改之前 vs 改之后**：
  ```
  改前：根据参考资料，纳税人转让旧房可按发票金额每年加计5%扣除。检索结果显示，持有满10年的按50%计算。图片链接 https://oss.example/t.png
  改后：纳税人转让旧房可按发票金额每年加计5%扣除。持有满10年的按50%计算。
  ```
  —— 只剪掉了"根据参考资料，/检索结果显示，/图片链接/那条链接"这些**字面检索腔帽子**，而事实数字（5%、10年、50%）一个字没动。**所以这 60 条是"我确定它一定去了痕"的标准对照。**
- 再让同一套新 Kimi 评分（同样每条打 3 次取平均）重读这 60 条。
- 结果：去痕度**平均涨了 +0.038**。这个 +0.038 可信吗？我用一个办法估它的不确定性：**把这 60 条反复随机重抽很多遍、每遍重算一次平均涨幅**（统计上叫 bootstrap），看这些涨幅散在哪个范围——得出**真实涨幅有 95% 的把握落在 [+0.005, +0.074]，整段都是正的、不跨过 0**。意思是：**再保守它也是涨的（最差 +0.005）→ 这套 Kimi 评分确实能读出"痕迹被去掉了"这件事。** ✅

**step111 · 实验B：再看"模型真训一遍，能不能自己学会去痕"**
- 为什么：Kimi 评分已经证明能读（实验A），现在拿它量"真训"的效果——这回读不出，就只能怪模型/数据，不能再赖评分。
- 做法：拿真用 mini DPO 训过的模型，让同一套新 Kimi 评分量它相对 base 的去痕度变化。
- 结果：去痕度**平均只涨 +0.0005**。同样用"反复随机重抽样、重算涨幅"的办法估不确定性，得到**真实涨幅 95% 把握落在 [−0.012, +0.013]——这个范围跨过了 0**。意思是：**真实涨幅可能是正、也可能是负，这点"提升"和"完全没提升"在统计上根本分不开 → 读不出 → 模型其实没学到东西。** ❌

> **怎么读这两个 95% 范围（这是分胜负的关键）**：范围**全是正的、不含 0**（实验A 的 [+0.005,+0.074]）= **确定涨了**；范围**一旦跨过 0**（实验B 的 [−0.012,+0.013]）= **可能涨可能跌、读不出**。同一套 Kimi 评分、同样每条打 3 次取平均，**A 读得出、B 读不出，差别只在"被测的那个东西到底动没动"**——评分是同一套好评分，动的是模型有没有真去痕。

**③ Kimi 的角色**：这一阶段 Kimi 是主评分——**一套新的去 RAG 专项评分**（只判 去痕度 trace_free / 守依据 grounded / 准确率，**每条独立打 3 次、取平均**来降噪）。提示词全文：
```
[系统] 你是严格、客观的中文税务问答质检专家。只输出 JSON。
[模板] 本任务不要求判断"是否像人类推理"，只评三项。【问题】{query}【参考资料】{reference}【标准答案】{gold}【think】{think}【answer】{answer}
1. accuracy：answer 与标准答案是否一致，0~1。
2. trace_free：think 是否没有 RAG/检索/参考资料暴露痕迹，0~1。高分=自然说依据但不暴露"参考问答对/资料显示/问题1/图片链接"；
   低分=出现上述字眼或大段照搬政策原文。不要因为"不像真人"扣分，只看去没去 RAG 痕迹。
3. grounded：think/answer 的事实数字政策是否被参考支持且不矛盾，0~1。
4. rag_traces：["explicit_ref","verbatim_copy","ref_enumeration","policy_source"]。
只输出 JSON：{"accuracy":"...","accuracy_score":0.x,"trace_free":0.x,"grounded":0.x,"rag_traces":[...],"comment":"..."}
```

**④ 规则的角色**：这一阶段规则只剩**一个程序函数 `trace_surgery`**（上面那 11 条正则，机械剪掉检索腔帽子和图片/链接），专门用来给 Kimi 评分造"已知去了痕"的标准对照。**除了实验B那一次 mini DPO，这步没有训练、没有 reward。**

**⑤ 结果 / 错在哪**：**新 Kimi 评分能读出去痕（实验A +0.038，95% 范围全正）、但模型真训没学出来（实验B +0.0005，95% 范围跨 0）。** 再加上第三条旁证——**模型对一道题采样 8 次、取这 8 个里最高分的那个、再跨全部题平均，也只有 0.701，和基座 0.697 基本一样**（这个 0.701 是 corrected-v3 这一轮的"全量未过滤"测法）。
> 这里和阶段4 的"chosen 均值 0.787"**不矛盾，是两个不同的数**：阶段4 的 0.787 是 corrected-v2 那一轮、而且**只统计"过了 ≥0.70 门"的好样本**（被筛过的高分子集，所以均值高）；这里的 0.701 是 corrected-v3 这一轮、**每题最高分跨全部题平均、没有任何过滤**（原始全量，所以低）。一个是"过滤后高分子集"、一个是"全量原始最高分"，加上是两轮不同的 run，数值不同很正常。
- **三条合起来：当时模型采样里根本没有可学的好样本。** 问题这次不在 Kimi 评分（实验A 已证它能读），在**模型采样本身**。
- **但"实验A 证明 Kimi 能读"这句要小心别读过头——这里点破一个一直没量清的东西。** 实验A 喂给 Kimi 的是"机械去痕"：整段检索腔帽子被剪掉，**是一个很大、很明显的改动**，Kimi 当然读得出。这**只证明它能分辨"大改动"，没说明它的分辨率有多细**。而"RFT → mini DPO"真训带来的差距是**很细**的一点——很可能 Kimi 分得清"明显脏 vs 明显干净"，却分不清"挺干净 vs 再干净一点点"。**所以"Kimi 能测出来"只到"能测出大改动"为止；它到底能稳定分辨多小的差，这一步压根没量——这正是下一波实验要先量清的（见收尾）。** 这也回到那条铁律：得先知道"预期能涨多少"和"Kimi 能分辨多小"，两个数摆一起才知道值不值得训。
- **另一个从头压到尾的负担：我们一直让 Kimi 一次评 3~4 件事**（答案对不对 + 守不守依据 + 去没去痕），对单次调用要求太高、几件事互相干扰。**直到阶段8 才把活拆开**——精确的"答案漂没漂"交确定性规则，模糊的"think 干不干净"才交 Kimi，一个裁判只判一件事，质量才稳。
> 走到这我本想放弃自采样强化学习，但**项目硬约束是"强化学习必须走通、DPO/GRPO 都保留"**，于是有阶段 6。

## 阶段 6 · derag_v4 · 我先用 Kimi 造好样本再训（三阶段），跑通了还是没涨

**① 我在做什么**：既然采样里没好样本，先用 Kimi 把好样本造出来种进模型，再训。三段：注入 SFT → DPO → GRPO。

**② 数据流（编号 123→127）**：
- **step123 基线推理**：**RFT 基座**贪心答 **2015 训练题** → `123_rft_train_greedy` 2015。
- **step124 挑出"还带痕迹"的题、让 Kimi 改写**：RFT 基座答完 2015 题后，**用确定性规则挑出思考里还残留念手册痕迹的题**——满足任一条就算：① 痕迹条数 >0（出现"参考资料/编号罗列/孤立甩文号"等）；② 遮掉税务事实后的照抄率 >0.35（先把税率/文号这类必保事实盖住，再看它和参考资料字面重合多少，还高就是真照抄）；③ 有客服话术（"小贴士/您可以参考下图"）；④ 有问答体（"问题1：…回答：…"）。**从 2015 题挑出 737 题** → 让 **Kimi 改写工**（温度 0.3）把这 737 条思考改干净 → `124_rewrites` 737 条。
- **step125 筛改写、注入微调出 s1**：拿这 737 条改写，**先过一道确定性硬规则（代号 L0）**，把明显不合格的直接刷掉——格式坏、引入了参考里没有的新数字、还带"参考资料"显式字眼、还在编号罗列、有图片/图床链接、照抄率 >0.55、整段重复废话。**737 条刷掉 83 条、剩 654 条进入下一关。** 下一关本想让 **Kimi 做二值判断**（每条只回答"干净/还脏"），但先拿一批"已知好(60 条)+已知坏(30 条)"的标准样本测了下它准不准——**判好/坏的平均正确率只有 0.6333**（满分 1、瞎猜 0.5，等于勉强比瞎猜好一点），不达标（要 ≥0.85）→ **判定 Kimi 这道门不胜任，降级：直接拿 L0 规则的结果当门**（`125_gate_rewrites.json` 标记 DEGRADED-GO）→ 留 **675 条当注入种子 + 76 条留作早停** → 监督微调出模型 **s1**。
  - **s1 注入完在 224 验收题上评了一下**（用一个冻结的确定性痕迹计数器，它的问题下面 ⑤ 会说）：无痕题占比 0.241、痕迹总数 127、答案准确率 0.70——**和注入前基本一个样，没看出注入把痕迹压下去。**
- **step126 先确认"池子里有没有好料可学"再往下训**：注入出 s1 后，让它对 737 题**各采 16 遍 = 11792 个候选**，数一下**有多少比例的题"至少有 1 个干净候选"**——结果 **76.8% 的题有干净候选**（这道检查我叫它"正样本密度检查"），够了 → 放行往下训。
- **step127 DPO 构对**：输入 = 737 query 的候选 → **规则构对** → **244 对** → train 204 + heldout 40 → DPO 出 **s2**。
- **GRPO**：**s2** 在 2015 题上在线 K=16 采样 + `derag_reward` 打分 → **s3**。
- 三个 run：`022656`（第一次 NO-GO）、`135121`（第二次 NO-GO）、`151520`（跑完，净≈0）。

> **一个自然的疑问：阶段5 不是证明了 Kimi（打 3 次取平均）能读出去痕吗？这里为什么不直接用它来审核每条改写，反而又搬出规则 L0 门 + Kimi 二值判断？**
> 因为这是**两件难度不同的事**：阶段5 证明的是"**给一整批样本打个总分、读出平均去痕 +0.038**"（聚合判断，单条的波动会在几十上百条上互相抵消）；而这里要做的是"**逐条给每一份改写盖章：够不够干净、能不能进 SFT 种子**"（**逐样本的二选一判断**）。**单条卡在一个阈值上判过/不过，波动消不掉**——Kimi 单条打分（哪怕打 3 次取平均，标准差仍 ~0.056）在阈值附近会大量判错（在那批标准样本上的好坏排序能力只有 0.69）。我先试"打 0~1 连续分逐条判"（毛病见阶段5），再退一步试"只判干净/还脏的二值判断"（以为粗颗粒更稳），结果二值也不胜任（判对率 0.6333），最后只能降级用 L0 规则兜底。**你的直觉是对的——这确实是 Kimi 评分质量的一次倒退；根因是"给整个模型打个总分"能过关的评分，未必扛得住"给每一条改写单独盖章"这个更细的活。**

**③ Kimi 的角色**：**改写工**（124，复用阶段 1）+ **二值门禁裁判**（125）。门禁裁判提示词全文：
```
[系统] 你是税务领域 RAG 痕迹审查员。只做二值判断，必须先指认证据，只输出 JSON。
[模板] 你只回答：待审 think 是否残留机械 RAG 痕迹？不判答案对错、不打连续分。
【问题】{query}【参考材料】{reference}【待审 think】{think}【辅助特征】masked_copy/citation_density/残余匹配提示
痕迹类型 A 提及检索装置；B 客服话术；C 答案体复刻；D 清单式甩文号；E 无消化照搬。
以下永远不是痕迹：必要税率/日期/金额/法定术语、会计分录、表单栏次、操作路径、官方网址、嵌进真实推理的法规名或文号、
        问题本身要求的税率表/清单、为保证答案正确必须保留的事实、单纯篇幅长。
有痕迹→列 trace_spans(逐字≥6字)、verdict=traced；无→trace_spans 空、verdict=clean；拿不准 confidence=low。
只输出 JSON：{"trace_spans":[{"span":"逐字片段","type":"A-E"}],"verdict":"clean|traced","confidence":"high|low"}
```

**④ 规则/奖励函数的角色**：这一阶段规则干三件事——(a) step124/125 的种子筛选（上面已说）；(b) step127 挑 DPO 对子；(c) GRPO 的奖励函数。后两个**先用大白话说在干嘛、再给完整公式**。

**GRPO 奖励函数**（`reward_v3.derag_reward`）核心思想："答案别崩、别照抄、念手册痕迹越少，给分越高"。先设几道红线（踩了就给很低分）：格式坏/重复废话给 −1；答案关键事实丢了或编了新数字，只给 0.1；照抄太狠（>0.55）或带图床链接，只给 0.1。都没踩，就按"痕迹越少给分越高"给 0.3~1.0：
```
masked照抄率 = 把税务事实(税率/文号/法定术语/会计科目/表单名/日期)先遮成□，再看 think 和参考资料字面重合多少
痕迹扣分 = 0.34×显式引用次数 + 1.00×编号罗列次数 + 0.20×孤立甩文号次数
         + 1.20×max(0, 甩文号密度−0.15) + 2.00×max(0, masked照抄率−0.25) + 8.00×max(0, masked照抄率−0.40)
最终奖励 = 0.3 + 0.7 × [ 1/(1+痕迹扣分) ]      # 扣分越多→括号里越小→奖励越低
```

**DPO 挑对子**（已把阶段3 那个"答案信号和思考信号串台"的硬伤修掉了——这次好坏只比痕迹、不比答案，正是"答案别掺进来、信号全留给 think"的拆法）：用大白话说就是——
- **好样本**：一处念手册痕迹都没有、遮事实后照抄率 ≤0.30、答案关键事实召回 ≥0.75、答案没跑偏（≥0.55）；
- **坏样本**：至少有一处痕迹、但答案也没崩太多（≥0.45）；
- **好坏之间差距要够明显**：痕迹条数差 ≥2，或差 ≥1 且照抄率差 ≥0.12（差太小就不成对）。

**⑤ 结果 / 错在哪**（两次卡死的原因 + 致命发现）：
- **第一次 NO-GO**：Kimi 改写的好种子过不了门——**不是改写差，是 L0 那道硬规则误杀了约 75%**。误杀来源逐条说：照抄率卡在 0.30 这道窄线、把"按 13% 税率"这种必保事实的字句重合也当成照抄（误杀 116 条）；同一处照抄被两条规则各扣一次（重复扣 177 次）；把"多引了几个文号"的合法推理句当成乱甩文号（误杀 174 次）；把会计分录"应交税费—应交增值税"这种天然重复当成废话退化（误杀 312 次）；多个指标取最严的那个、只要一项偶尔偏低就全盘否掉（误杀约 60%）。
- **第二次 NO-GO**（第二个 run `135121`，那时 Kimi 门用的是"打 0~1 连续分"那种）：我拿一批"已知好(60 条)+已知坏(30 条)"的标准样本（这批样本我叫它"锚集"），测这套 Kimi 评分能不能**把好的排在坏的前面**——**这个排序能力（AUC，满分 1、瞎猜 0.5）只有 0.6878**，很差。原因：我让它"先找证据再打分"，结果它**对每条都挑毛病、把好样本也一起压低**（好样本平均分才 0.548），好坏混一起分不开。（第三个 run `151520` 改成"只判干净/还脏"的二值判断，仍不胜任，那个 0.6333 就是它判对的比例。）
- **第三次跑完，净≈0**；**最致命**：评测用的那个冻结痕迹计数器里，**痕迹总数 127 条中有 76 条（60%）其实是合法政策引用**（"按《印花税法》第四条"这种），根本不是念手册——**等于在优化一个六成是误判的指标**（这条在阶段7 详查）。
- **一句话定性这一阶段**：它**始终没解决"Kimi 逐条打分太抖、做不了逐条筛选"这个根问题**（连续分、二值都试过，都没扛住），只是**降级用 L0 硬规则兜底、硬把三阶段链路推完**，结果证明——评分没修好就硬推，确实没用（净≈0）。

## 阶段 7 · reward v5 · 我把"被优化的指标"逐条核对，发现它九成是假的

**① 我在做什么**：不再瞎训，人工逐条核对那些被判"痕迹"的样本。

**② 数据流**：输入 = 138 报告里被判痕迹的样本 → **人读 + 正则双判**（无新 Kimi、无新 reward）→ 诊断结论。

**③④ Kimi / 规则**：都退场，纯人读 + 正则核对。

**⑤ 结果 / 错在哪**：
- **人工审了 96 条被规则标成"政策痕迹(policy_source)"的样本：0 条是真痕迹，96 条全是合法引用法条**（如"按《印花税法》第四条规定"）——和规则的判定完全相反。
- **而且引法条的答案反而更准**：含这类政策引用的回答，答案正确率比不含的略高 +0.03~0.05（即"政策引用出现"与"答案分"的相关系数为正）→ **去压制它，等于在压制正确推理，方向反了。**
- 剔掉这些误判后，**真痕迹只剩 22/224 = 9.8%。**
- **"24% 干净"是个误导数字**：它其实是阶段 6 那个 **"够不够格当 DPO 好样本"的通过率**（要同时满足：一处痕迹都没有、遮事实后照抄率 ≤0.30、答案关键事实召回 ≥0.75、答案没跑偏 ≥0.55），**根本不是"无痕率"**。真正的无痕率是 **0.74~0.89**——**224 题大半本来就干净。**
- 评测噪声门槛 **MDE=0.116**（这套二值"干净率"在自己噪声下、至少要涨 0.116 才读得出）——真信号小一个数量级——**四轮"净≈0"很大程度是根本测不出。**

## 阶段 8 · 探针 + X 重测 · 我花 1 天不训练，先确认"到底有没有救"

**① 我在做什么**：烧卡前先回答"那批病题有没有救"。先把答案标准改对——**不追求"答案对"，只追求"不漂移 V1 自己会给的答案"**（V1 每题采 8 次建"认可范围"，新答案结论极性被覆盖=没漂）。

**② 数据流（编号 150→159b）**：
- **step150 挑病题**：输入 = **RFT-merged 基座**贪心答 **224 验收 + 1000 训练 = 1224 题** → **Kimi 去 RAG 评分对每条打 2 次取平均**来挑病题（去痕度 <0.70、或被标出结构性痕迹的，算病题）→ **从 1224 题里挑出 489 道病题**。（注意这个"打 2 次"是同一条重复判分降噪，和下面 step151 的"采样 16 遍"是两码事。一开始我用规则挑，只挑出 22 道，漏太多。）
- **step151 自采样**：**RFT-merged 基座**对 489 病题各 **K=16**（温度 0.9）→ `151_rft_samples` 489 题×16 遍。
- **step152 V1 库**：**原始 V1**（系统提示 = RAG 腔 SYSTEM_PROMPT）对 489 病题贪心 1 遍（温度 0）+ 采样 8 遍（温度 0.8）→ `152_v1_support` 489。
> **先说清"判一道病题自救成功"要同时看两件事**：① **思考干不干净**（去没去念手册腔）；② **答案漂没漂 V1**（结论/数字在不在 V1 认可范围）。16 遍里只要有 1 遍两件事都过 = 这题自救成功。下面 step153 和 step153b 的差别，**只在"思考干不干净"那件事换了谁来判**，"答案漂没漂"两步都用同一把规则。

- **step153 规则判（得 X_rule=0.851，但虚高）**：两件事都用规则判——
  - **答案漂没漂** = 规则 `answer_in_support`：机械比对"这条答案的结论极性（免征/应缴…）和关键数字，在不在 V1 那 8 次采样覆盖的集合里"。**这是确定性事实比对，可信。**
  - **思考干不干净** = 规则 `real_trace`（显式引用+编号罗列+真逐字照抄，剔掉合法政策引用）：**只会数字面痕迹词，对"换了词的结构性念手册"全瞎，把脏的也判成干净 → 这一项虚高。**
- **step153b Kimi 重判（得 X_kimi=0.791，才是真的）**：**只把"思考干不干净"这一步从规则换成 Kimi 判**（阶段5 那套去 RAG 评分，每条打 2 次取平均）；**"答案漂没漂"仍然用规则 `answer_in_support`**（复用，因为它本来就可信）。为省 Kimi 调用，先用规则把"答案漂了的"过滤掉，只对"答案不漂"的样本让 Kimi 判思考。
  - **X_kimi=0.791 是怎么算出来的**：**489 道病题里，有 387 道做到了"16 遍里至少 1 遍 思考被 Kimi 判干净 且 答案没漂"，387 ÷ 489 = 0.791。它是"自救成功的题数占比"，不是把一堆分数取平均。**（X_rule=0.851 同理：规则判法下 416 道自救成功 ÷ 489；只是这里的"思考干净"是规则判的、几乎不卡人，所以 416 偏多、X_rule 偏高。）

**③ Kimi 的角色**：挑病题（150）+ 在 153b **只判"思考干不干净"这一件事**（复用阶段5 去 RAG 裁判，每条打 2 次取平均）。**它全程不碰答案。**

**④ 规则的角色（同一类规则、两件事两种命）**：
- **判"答案漂没漂 V1"（`answer_in_support`）= 可信，153 和 153b 两步都用它**——因为这只是比对答案的结论极性/数字在不在 V1 范围，是确定性事实活，不需要"看懂结构"。

- **那 Kimi 能不能判答案？能，但没必要**：Kimi 的 accuracy/grounded 维度本来就在评答案。但"答案漂没漂 V1"是个**精确的事实比对**（结论极性/关键数字在不在 V1 那 8 次采样的集合里），**确定性规则做得又准又便宜、零噪声**；Kimi 判这种事反而引入抖动（它当硬门的 balanced_accuracy 也只有 0.633）。所以分工是：**精确事实活（答案在不在 V1 范围）交规则，模糊语义活（思考是不是念手册）交 Kimi**——各用各擅长的。

**⑤ 结果 / 错在哪 + 下一步**：
- **规则判得 X=0.851 为什么不可信**：因为规则判"思考干不干净"几乎没有判别力——① 489 病题里 98%（479 道）规则数出来的痕迹是 0，却照样判"干净"；② 自救失败的 73 道里，72 道是栽在"答案漂移 V1"（这个规则判得了）、只有 1 道栽在"思考脏"（这个规则看不见）。**所以规则那道"思考干净"的关形同虚设，X=0.851 实际测的是"答案没漂 V1"、不是"思考干不干净"。**（出处：108 号文档）
- **Kimi 重判得 X=0.791**（全体；eval 0.779 / train 0.794，两边接近、不是过拟合）。

  > **核心结论：79% 的病题，模型 16 次采样里存在"思考干净 + 答案不漂"的候选——可学的好样本真实存在。**

  - **这个 0.791 信到什么程度**：单条 Kimi 打分有波动（同一条反复打、标准差 ~0.056），但 489 道题取个比例、波动互相抵消，所以**可信的是"整体自救率约 0.79 这个量级（已明显低于规则虚高的 0.851）"，不必当真小数点后的 0.791**。另有旁证它没乱来：我另跑了 5 个独立裁判复核 18 条"规则判干净"的样本，他们判的真干净率 ~0.44，和 Kimi 的逐条 ~0.42 对得上。
- **把阶段 6 和这里拼起来（全书的关键矛盾）**：好样本真实存在（Kimi 判 X=0.791），**但阶段 6 用字面规则当 GRPO 奖励，一道好样本都没能利用上（实测净≈0）**——因为那个规则奖励只看字面痕迹词、对"换了词的结构性复述"全盲，而且 16 遍里 13.5 遍都被它判成干净（分数饱和、没有梯度可学），GRPO 于是钻空子、把模型优化成"改头换面接着复述"。**所以瓶颈钉死在"奖励函数"，不在"有没有好样本"。**
- **下一步（最优先：先把两个打分工具的能力量清楚，再烧卡）**：动手训之前，先用一批"已知干净 / 已知脏、且差距可控"的标准样本，把两件事标定出来——㈠ **Kimi 的分辨率**：它能稳定分辨多大的去痕差（0.05？还是 0.02？），低于这个差的提升它根本读不出、也就不值得训；㈡ **规则的盲区边界**：哪些"换了词的结构性念手册"它必漏、哪些合法引用它必误杀。把这两个工具"能测什么、测不出什么"标清，后面所有"涨没涨"才有可信判据。**在此基础上**再走：① **用 Kimi 离线标 DPO 对**（chosen=Kimi 判干净 / rejected=Kimi 判脏，离线标所以不怕在线训练时被钻空子）；或 ② 做一个**确定性的"语义照抄率"**（看思考和参考资料的语义重叠，而不只是字面重合）。另：**15%（72/489）的题答案 16 遍全漂 V1**，是个独立的"守依据"待办。

## 阶段 9 · judgecal · 先把 Kimi 这把打分尺校准——它到底能分辨多大的"换词复述"差

**① 我在做什么**：承接阶段 8 结尾那条铁律。规则是确定性的、不用标；**要标的是 Kimi**——我要拿一批"已知脏到什么程度"的标准样本，量出 **Kimi 能稳定分辨多大的"换词复述"差**。低于这个分辨率的提升，它根本读不出、训了也白训。

**② 数据流**：
- 造梯子：拿 **13 个参考资料**，每个都人工写出 **6 档脏度** 的 think——**没抄 / 抄 1 句 / 抄 2 句 / 抄 3 句 / 抄 4 句 / 完全照抄**（"抄"指换词复述：把参考某段换个说法重讲一遍）。13 × 6 = **78 条 think**（每档 13 条）。
- 反复打分：让 **Kimi 给每条打 0-10 干净分、独立打 16 遍**（同一条打 16 次，为的是把"同一条反复打的抖动"和"不同样本之间的差"两种波动拆开看）→ 78 × 16 = **1248 次打分**（`judgecal/main/162_judgecal_report.md`）。
- 另设一条"客观锚点"（人为定的理想分标尺：没抄=10、每多抄一句−2、完全照抄=0），**只当对照线**，判两档分不分得开完全不靠它、只看 Kimi 的实际打分分布。

**③ Kimi 的角色**：**0-10 整段干净分评分员**（第一次出现，提示词全文，`judgecal_common.py:106-132`）：
```
[系统] 你是税务 think 的『换词复述照抄』程度评分员。给整段 think 打一个 0-10 的『干净分』：
       完全没有换词复述照抄=10 分，整段几乎都是把参考逐条换词复述=0 分。
       不评文风长短、不判答案对错，只看『有多少是把参考某段换个说法复述了一遍』。只输出 JSON。
[模板] 给你一道税务题的【参考资料】和一段【think】。请给这段 think 打一个 0-10 的『干净分』——
       分数越高=越没有"把参考某段换个说法重新复述一遍"；分数越低=越多句子在逐条复述参考。
评分锚点（共 6 档，照抄越多分越低）：
- 10 分：完全没照抄——全是从问题自己一步步推，或只是把参考里的事实(税率/金额/期限)自然用进推理(这【不算】照抄)。
- 8 分：只有 1 句是把参考某段换词复述。   - 6 分：有 2 句换词复述。
- 4 分：有 3 句换词复述。               - 2 分：有 4 句或更多换词复述。
- 0 分：整段几乎每句都是把参考逐条换词复述(完全照抄)。
关键：把参考里的某个事实自然用在自己的推理里【不算照抄】；只有"整句在复述参考某段"才算。不要因为篇幅长/用词正式/含专业术语就扣分。
【参考资料】{reference}【think】{think}
只输出 JSON：{"clean_score":0到10之间的数字,"n_copied_est":你估计的换词复述句数,"reason":"一句话理由"}
```

**④ 规则的角色**：只有那条"客观锚点"（人为理想分标尺），**只当对照线**，不参与判可分。

**⑤ 结果 / 这把尺到底能干什么**：把每条打 16 遍的结果拆成两块波动——
- **σ_judge = 同一条反复打的抖动**（多打几次取平均能压掉）；
- **σ_between = 同一档里不同样本，Kimi 自己就给得忽高忽低**（多打也压不掉，这是真天花板）。

**第一件事 · 整体评测（给一个模型阶段打总分）够用，但要靠大 N。** 评一个模型整体干净度 = 让 Kimi 给 N 条各打 k 次取总平均，这个平均值的标准误（衡量"这个总平均本身有多准"）：

| 样本量 N | k=1 | k=3 | k=16 |
|---|---|---|---|
| 224 | 0.084 | 0.074 | 0.069 |
| **500（V2 验收集）** | **0.056** | **0.050** | 0.046 |

- **两个模型阶段的平均干净分，差距要 > 约 3×SE 才算真涨（不是噪声）。N=500、k=3 时 SE≈0.050 → 两阶段差 > ~3×0.050 ≈ 0.15 才可信。** ——**记住这个 0.15，它就是后面阶段 12 判 DPO"真涨没"的那道门。**
- **多打几次（加大 k）几乎不改善整体评测**（N=500 时 k=1→16 只把 SE 从 0.056 压到 0.046）——因为整体评测的瓶颈是 σ_between，它靠 √N 摊薄、不靠 k。**所以整体评测用大 N、小 k（k=1~3）就够，别浪费 16 倍调用。**

**第二件事 · 逐条选对子（判单独一条够不够干净）很难，σ_between 压不掉。** 把每两档算"|均值差| ÷ (两档 σ 之和)"，**≥3 才算两条 3σ 带不相交、可分**：中间档全分不开（没抄↔抄 1 句 = 0.8 ❌、抄 1 句↔抄 2 句 = 0.9 ❌、抄 2 句↔抄 3 句 = 1.1 ❌）；**只有"很干净（没抄，分≈7）"对上"重度照抄（抄 4 句/完全照抄，分≈0）"才够 3σ 可分**（没抄↔抄 4 句 = 3.8 ✅、没抄↔完全照抄 = 4.0 ✅）。

> **核心结论（这把尺校准的最终判词）：Kimi 这把尺，给整个模型阶段打总分够用（大 N、小 k），但逐条挑 DPO 对子只能做"明显干净（≈7）vs 重度照抄（≈0）"这种大间距对子，中间档一律分不开、不能用。** 这一条直接钉死了后面 DPO 对子的唯一造法（见阶段 12）：chosen = 洗干净的 think（≈7），rejected = V1 原版重度照抄的 think（≈1-2），中间档不要。

  - 三道门（缺一不可）：**① 规则门**（`detect_rag_style` 判无检索腔表面词）**∧ ② facts 极性门**（没引入参考里没有的新数字/新极性）**∧ ③ k=2 门**（让 Kimi 对"改写 vs V1 原 think"各打 2 次，改写得每次都判比 V1 干净）。

**③ Kimi 的角色**：**改写工（新提示词）+ k=2 选样门**。新改写法和旧改写法的差别，就是这一阶段能打穿天花板的根因，必须讲清——

**旧改写法（前八阶段一直用的，天花板 Kimi 干净分 3.44）**：把 V1 的 think 喂进去当"必须保留的要点清单"，还加一条"必须扣依据"逼 Kimi 把引文号改个说法。结果 Kimi 只能在 V1 的措辞上**换词**，等于**逼它做"换词复述"**，而且"逼改引文号"这条反而**自己引入了 policy_source 痕迹**（"按某文号"被改写得更像罗列）。——这正是阶段 7 发现的"九成假痕迹"的一个来源。

**新改写法（`rewrite_v2.py`，把天花板打到 7.11）**：**不喂 V1 的 think**，改成只给【用户问题】+【事实依据】+【必须命中的结论与数字（V1 认可范围，即池锚点）】，让 Kimi **从问题从头推**。系统提示加四条铁律（`rewrite_v2.py:28-33`）：
```
1.【不复述】绝不出现"把依据某段换个说法重讲一遍"的句子。
2.【不检索腔】全程绝不出现：参考问答对、参考资料、检索结果、资料显示…
3.【事实与结论锁死】推理里每一个税率/百分比/金额/数字/日期/期限、以及结论极性，都必须取自【事实依据】。
4.【宁简勿堆】宁可写得简短句句在推，也不要为了显得专业而堆砌。
```
**一句话区别：旧法喂 V1 思考→只能换词复述（天花板 3.44）；新法不喂、从问题从头推→真正自然推导（7.11）。**

**④ 规则的角色**：三道门里占两道——规则门 `detect_rag_style`（判表面检索腔）+ facts 极性门（防臆造新数字）。**这步没有 reward（是监督微调）。**

**⑤ 结果 / 错在哪**（在冻结 500 验收集上、三件套口径）：

| 阶段 | Kimi 干净分(k=3) | 规则去检索腔通过率 | 答案在池率 |
|---|---:|---:|---:|
| V1 baseline | 3.140 | 2.6% | 93.8% |
| **SFT-2s** | **4.408** | **45.4%** | **83.0%** |

- **核心结论：这是全项目第一次"双真涨"——Kimi 干净分（3.140→4.408，涨 1.27，远超阶段 9 那道 0.15 的真涨门）和规则去检索腔通过率（2.6%→45.4%，翻盘）同时真涨，而且不是 Goodhart（前八阶段从没让这两个一起涨过）。** 换改写法（治根）+ 选种子要够量（930 条），打穿了旧天花板。
  > 口径提醒：备忘里另有一处写"kimi_think 3.1→4.4、规则 think 3%→45%"，那是 SFT 当时的内部读数，和这张表的 k=3 整段干净分 `clean_mean` 数值几乎一致、但**严格说是另一口径的 Kimi 分**，别和别的链混着比。
- **代价（这步埋的雷）：答案在池率掉到 83.0%，跌破 0.85 地板（−10.8pp）。** 改写"只改 think、不动 answer"，按理答案不该动，但**微调时模型把"去痕的 think"学进去，顺带把答案分布也推偏了一点**。我没在这步硬卡 0.85，而是设了一条**放行线 `V2_PRUNE_INPOOL_FLOOR=0.80`**：只要在池率 ≥0.80 就放 SFT 进下一阶段，**指望 RFT 的答案门从自采样里把 grounding 捞回 ≥0.85**（见阶段 11）。**注意：0.80 只是中途放行线，最终验收要对的是 0.85。**

## 阶段 11 · V2 RFT · 用"2σ 带不相交"门，从自采样里只挑"答案没漂"的，把 grounding 捞回地板

**① 我在做什么**：SFT 把 think 打干净了，却把答案在池率打到 83%（跌破地板）。这一阶段做拒绝采样微调（RFT），**专门用答案门从模型自己的采样里只留"答案没漂 V1"的**，把在池率捞回 ≥0.85，同时别把 think 的干净度弄回去。

**② 数据流**：
- **自采样**：**V2 SFT 模型**对 1739 训练题各**自采 32 条**（32 这么多，是因为 Kimi 精度不高、多采才可能采出"明显比基线更干净"的信号）。
- **两道筛**：先用规则把格式坏/答案漂的刷掉（survivors），再让 **Kimi 对幸存候选 k=16 双评**（给"自采样 think"和"SFT 基线 think"各打干净分，算两者差 gap）→ 过 **2σ 门** 的留下。
- **2σ 门**（`v2_common.py:49-58` 的 `confident_cleaner()`）完整判据：
  ```
  gap = s_clean − s_dirty                                  # 自采样比基线干净多少
  可分(留下) ⟺ gap > n_sigma · ( σ(s_clean) + σ(s_dirty) ) # σ 查阶段9 judgecal 标定表
  函数默认 n_sigma=3.0；RFT/DPO/冷启动选样时显式传 n_sigma=2.0 → 得"2σ 桶"
  ```
  大白话：**只有当 Kimi 有把握（两条 2σ 带不相交地）判"这条自采样比 SFT 基线更干净"，才把它当好样本留下。** 2σ 桶比 3σ 桶松、量更多。
- **微调 + 合并**：选中的 2σ 桶训出 RFT adapter → **合并进底座** = **`v2-rft-2sigma-2s-merged`**（62G / 14 shards）。**这就是后面 DPO 的训练底座。**

**③ Kimi 的角色**：k=16 双评（给自采样和基线各打 0-10 干净分，喂给 2σ 门算 gap）。

**④ 规则/奖励的角色**：2σ 门（`confident_cleaner`，公式见上）+ 规则答案门（`answer_in_v1_pool`，只留答案没漂 V1 的）。**这步是 RFT，不是在线 reward。**

**⑤ 结果 / 错在哪**（冻结 500、三件套）：

| 阶段 | Kimi 干净分(k=3) | 规则去检索腔通过率 | 答案在池率 |
|---|---:|---:|---:|
| SFT-2s | 4.408 | 45.4% | 83.0% |
| **RFT-2s-2s** | **4.489** | **45.6%** | **85.0%** |

- **核心结论：RFT 把答案在池率从 83.0% 捞回到 85.0%（正好踩到地板），同时 think 干净度没退（Kimi 干净分 4.408→4.489、规则 45.4%→45.6%，都微涨）。** RFT 这一步主要是"用答案门修 grounding"，干净度本就没指望它再涨多少。
  > 分工沉淀（这是 V2 链的方法论）：**SFT 用 k=2 门要的是"量"**（k2-930 比 V1 干净就行、不那么严，种子要够多才学得动）；**RFT/DPO 用 2σ 门要的是"质"**（Kimi 有把握判更干净的才要）。
- 一个口径澄清：常被提到的 **885 / 478** 是后面 **DPO 对子的条数**（2σ 桶 885 对、3σ 桶 478 对），**不是 RFT 的训练行数**，别搞混。

## 阶段 12 · V2 DPO · answer-lock 大间距对子，借 AutoDL 跑（踩满工程坑），think 真涨但答案差 0.4 个点

**① 我在做什么**：RFT 底座上做 DPO，用阶段 9 校准结论造的"大间距对子"把 think 继续推干净，**同时用 answer-lock 把答案锁死别动**。

**② 数据流**：
- **造对子（885 条）**：用阶段 9 的唯一可用造法——**chosen = 洗干净的 think（Kimi 干净分 ≈7）+ V1 原版 answer；rejected = V1 原版的脏 think（Kimi ≈1-2）+ 同一个 V1 原版 answer**。这对子天然 3σ 可分（"干净 vs 重度照抄"正是阶段 9 唯一稳分得开的那种），中间档一律不用。文件 `dpo_pairs.2s-2s_2sigma.v2.jsonl`（**885 对**；另有 3σ 版 478 对）。
- **answer-lock（关键）**：一对里 chosen 和 rejected 的 `<answer>` **拼的是同一段 V1 原版答案，只有 `<think>` 不同** → DPO 的梯度全压在 think 上、不动答案（修掉了阶段 3 那个"答案信号和思考信号串台"的老硬伤）。ms-swift 原生 DPO 格式（`v2_paths.py:119-132`）：
  ```
  { "messages":[ {system}, {user},
                 {assistant: "<think>洗净版</think><answer>V1 原版</answer>"} ],   # chosen
    "rejected_response": "<think>V1 脏版</think><answer>V1 原版</answer>" }          # rejected
  ```
  本次实测验证：885 条里前 5 条 chosen/rejected 的 answer 逐字一致 ✅。
- **训练（在 AutoDL 借卡跑）**：本地 8×A800 当时被同事占用，临时借 **AutoDL 2×A800-80G**。走**路线 B（不合并权重、只产 LoRA adapter）**——因为 80G 盘装不下每次 CPU 合并出的 ~65G 全量权重。DPO 跑 **108 步**（885 ÷ 16 × 2 epoch ≈ 108），`train_loss≈1.114`，adapter = `checkpoint-108`（257M），下回本地再合并评测。

**③ Kimi 的角色**：**离线**给 chosen/rejected 打干净分（造对子时用），**训练时 Kimi 不在场**（DPO 吃的是离线偏好对，所以不怕在线训练被钻空子）。

**④ 规则/奖励的角色**：2σ 门 `confident_cleaner` 选 chosen（公式同阶段 11）；answer-lock 由代码保证（同一段 V1 answer 拼进 chosen 和 rejected）。

**⑤ 结果 / 错在哪**（冻结 500、本地合并后 vLLM 推理 + Kimi k=3 评，2026-06-22 17:09）：

| 阶段 | Kimi 干净分(k=3) | 规则去检索腔通过率 | 答案在池率 |
|---|---:|---:|---:|
| V1 baseline | 3.140 | 2.6% | 93.8% |
| SFT-2s | 4.408 | 45.4% | 83.0% |
| RFT-2s-2s | 4.489 | 45.6% | 85.0% |
| **DPO-2s-2s-2s** | **4.643** | **49.0%** | **84.6%** |

- **DPO 相对 RFT：Kimi 干净分 +0.155，刚好越过阶段 9 那道"3×SE≈0.149"的门 → 判 `DPO_TRUE_GAIN`（真涨、不是噪声）**；规则通过率也从 45.6% 涨到 49.0%。**这是 DPO/GRPO 这条线上，第一次量出"强化学习把 think 去 RAG 化"是真的能学到的。**
- **但最终三件套 FAIL，唯一卡点 = 答案在池率 84.6% < 85.0% 地板（差 0.4 个点）。** 按 500 题算约 423/500，RFT 是 425/500。
  > 为什么 answer-lock 锁了答案、在池率还是掉？**answer-lock 只锁"训练对子里的答案文本"，推理时模型是现吐新答案的**；DPO 把 think 推干净的同时，那个"洗干净 think"的策略顺带把答案分布也推偏了一点点。**所以 DPO 不是废实验——它给下一阶段 GRPO 提供了一个"think 更干净、但答案略越界"的起点。**
- **第437题这个真实边界坑（诚实记账，不藏）**：有一题（"员工 1 月入职，3 月发放工资 1.4 万元…"）模型在 think 里反复循环直到撞上 `max_new_tokens=1536`，没吐 `</think>`、没吐 `<answer>` = 真实的格式失败。**口径：原始输出一字不改，诊断成 partial think / 空 answer / format_ok=false；partial think 照常让 Kimi k=3 打分计入均值，规则 think 判失败，空 answer 判不在池；不重采样、不补 V1/gold 答案、也不把这题剔除，分母仍是 500。** `format_failure=1/500`，已自然反映在规则率和在池率里。重推这一题就算成功也只补 1 题、改不了 FAIL，所以不补跑。

**这一阶段踩满了工程坑——为展示工作量，原样列出（每条都验真过）**：

## 阶段 13 · V2 online GRPO · 当前进行中（代码已落地，正等服务器出结果）

**① 我在做什么**：DPO 证明 think 能学干净，但在池率差 0.4 个点跌破地板。这一阶段做在线强化学习（GRPO），**从 DPO 合并底座出发，把"答案在池 + 格式完整"做成硬约束，目标是保住 DPO 已拿到的 think 干净增益、同时把在池率拉回 ≥0.85**。

**② 数据流**：
- **建训练数据**（`step_v2_build_grpo_data.py`）：**只读 1739 训练题**（冻结 500 不进训练），每行写 `messages + qid/query/user_prompt/gold_answer + v1_answers_json`。**过滤掉"不可训练的题"**——如果某题的 V1 认可池里没有任何可比较的极性/数字/日期事实，那模型怎么答都没法判在不在池（答"是"被拒、答具体值也被拒），这种题剔除（标 `v1_answer_pool_trainable`）。
- **训练（复用旧跑通的框架，不新造）**：仍走 ms-swift `rlhf --rlhf_type grpo --use_vllm true --vllm_mode colocate`，保留"显存五连关"（TP=8、`vllm_gpu_memory_utilization=0.5`、`move_model_batches=16`、offload、ZeRO3）。两段：**warmup（只用规则+答案硬约束，30 步，不调 Kimi）→ CPU 合并 → online（加 Kimi k=2，90 步）→ CPU 合并 → 冻结 500 三件套评测**。先 warmup 用零噪声的硬约束把答案在池/格式拉回，再 online 上 Kimi 继续推干净。

**③ Kimi 的角色**：只在 **online 阶段**用，**k=2、只做"组内排序"**——对同一题一组候选里、**已经过了答案/格式硬门的那些**，用 Kimi 干净分排个高低当奖励微调信号，**不是硬门**（默认对"规则 think 已失败"的样本不调 Kimi，省钱且不让噪声进硬门）。

**④ 规则/奖励的角色**：在线 reward 是一棵**词典序的硬门决策树**（`grpo_reward_plugin.py`，给全公式），**越靠前的门越不可被后面补偿**：
```
格式坏（<think></think><answer></answer> 不齐）或 think 太短(<40字)  → -4.0
答案不在 V1 池（answer_in_v1_pool 判漂）                          → -3.0   ← Kimi 永远补不回来
答案在池但"不可比较"（无极性/数字/日期可比）                      → -2.0
以上都过 → 看 think 干不干净：
   规则判有检索腔痕迹(detect_rag_style)  → 0.50 + 0.20·kn − penalty
   规则判无痕迹                          → 1.00 + 0.40·kn − penalty
   其中 kn = Kimi 干净分 / 10（归一到 0~1）；penalty = 重复罚 + 超长罚
```
**核心设计（回应阶段 6/7 的教训）：`answer_in_v1_pool` 是 Kimi 不可补偿的硬门**——再干净的 think 也救不回一个漂出 V1 池的答案。**绝不能像 derag_v4 那样"只用一个去检索腔指标当 reward"**（那会复现阶段 6 的 Goodhart：模型把痕迹词换个说法接着复述、还顺手把答案弄漂）。

**⑤ 当前状态 / 下一步**：
- **代码已落地，本地 23/23 测试通过**，正等服务器实跑出结果。
- **首跑又踩两个坑（继续展示工作量，都已 hotfix）**：① **22:39** —— `swift.grpo_reward_plugin` 和已安装的 ms-swift 包名冲突 → 改成**按文件路径加载** `code/swift/grpo_reward_plugin.py`、不新增 `swift/__init__.py`，避免污染 ms-swift 正常导入；② **22:43** —— 旧的 1739 行 GRPO 数据里混着"不可训练 answer pool"的题，审计正确报错 → builder 过滤该类题，runner 发现旧数据缺 `v1_answer_pool_trainable` 字段时**自动改名 `.stale-unfiltered-*` 并重建**，不用手工删。
- **验收口径**：GRPO 跑完仍在冻结 500 上对三件套——Kimi 干净分不能回落到 RFT 的噪声带内、规则通过率不低于 DPO/RFT、**答案在池率必须回到 ≥0.85**。

**全书一句话**：这个项目不是"强化学习失败"，而是我一次次发现——**手里两个打分的东西轮流没对准目标**：Kimi 打分波动大、当"逐条硬门禁"时不稳（和真值相关只有 0.077、好坏排序能力只有 0.69）；本地奖励函数只认字面、看不见换了词的念手册、还把合法政策引用当痕迹（60% 是误判）。**四轮强化学习的"没涨"，每次拆开都是"打分/选样的标准没对准"，不是强化学习本身不行。最近终于证明"其实有救（好样本占 0.79）"，剩下的坎只有一个——把奖励函数换成一个"看得见语义、又不被模型钻空子"的打分法。而动手换之前，得先把现有这两个打分工具"能分辨多大的差"量清楚——这是下一波实验的第一件事。**

**这下一波，就是 V2/derag2 链（阶段 9~13），到目前为止它把前面的账一笔笔还上了**：先（阶段 9）把 Kimi 这把尺标定清楚——它给整个模型阶段打总分够用（大 N、小 k），逐条选样只能做"很干净 vs 重度照抄"的大间距对子；据此（阶段 10）换了改写法（不喂 V1 思考、从问题从头推）做 SFT，**拿到全项目第一次"双真涨"**（Kimi 干净分 3.14→4.41、规则去检索腔 2.6%→45.4% 同时真涨、非 Goodhart）；（阶段 11）RFT 用答案门把在池率从 83% 捞回 85% 地板；（阶段 12）DPO 用 answer-lock 大间距对子把 think 继续推干净，**第一次量出强化学习的真涨**（Kimi 干净分 +0.155 越过 3×SE 门 = `DPO_TRUE_GAIN`），可惜答案在池率 84.6% 差 0.4 个点没守住地板、三件套仍判 FAIL。**所以到"现在"，结论已经从"强化学习能不能学到东西"变成了"能学到（DPO 真涨了 think），只剩'别把答案弄漂'这一个工程约束没收口"**——这正是阶段 13 GRPO 要做的事（把答案在池做成 Kimi 不可补偿的硬门），代码已就位，**全书停在"等服务器跑出那张三件套表"。**
