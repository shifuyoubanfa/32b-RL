# 新版实验设计（技术方案V2）· 模块1：judgecal —— 标定 Kimi「换词复述照抄」的分辨率

> V2 的**第一个、也是必须先跑的模块**。不训练、不碰 GPU、只调 Kimi。
> 一句话：**训练之前，先量清楚 Kimi 这把唯一要靠的尺子，能把"照抄多少"分到多细——它的分辨率上限，就是整个项目的强化学习上限。**

---

## 0. 为什么先做这个

V1 四轮强化学习都"没涨"，根子是**打分/选样的工具没对准目标**：规则只认字面、看不见换了词的念手册；Kimi 当逐条门禁又太抖。阶段8 已证**可学的好样本真实存在（自救率约 0.79）**，瓶颈钉死在"打分工具"上。所以 V2 铁律：**先把工具能力量清楚，再训。**

---

## 1. 目标缩窄（前提）

> **think 链不应出现"明显的换词复述照抄"——把参考资料某段换个说法、逐条复述了一遍。**

分工，互不重叠（**规则不碰照抄；照抄一律 Kimi**）：

| 目标对象 | 谁管 | 可靠性 |
|---|---|---|
| 答案漂没漂 V1 | 规则 `answer_in_v1_pool` | 确定性、零噪声 |
| think 检索腔表面标记 | 规则 `detect_rag_style` | 确定性、零噪声 |
| think **照抄（原样 + 换词复述，所有形式）** | **只能靠 Kimi** | 语义判断、规则做不可靠（阶段7 误判）→ **本模块标定它** |
| think 彻底重构、看不出搬运 | 谁都不管（人也看不出） | 不在范围 |

> 规则两个函数落在 `code/pipeline/rules_v6.py`。judgecal 里也对每条 think 跑一遍 `detect_rag_style` 作旁证——这些 think 是"照抄但不带检索腔词"，所以规则应一律判 `has_rag_style=False`，照抄全靠 Kimi（见 step162 报告的"规则检索腔命中"行）。

---

## 2. 本质是一个分辨率问题（这模块的核心思路）

"完全照抄 / 部分照抄 / 合法用事实 / 没照抄"这些类别，**本质就是带语义的精度档位**。我们要测的不是"Kimi 能不能逮到照抄"（那只是能不能分开最两端），而是：

> **Kimi 能把这些档位分到多细？它的打分曲线在哪一段"趋于直线/压平"？那个拐点 = 它的分辨率上限。**

把档位用数字锚定，排成由干净到脏的 6 档（部分照抄按句数拆开）：

| 真实档位 | 客观锚点(干净度) | 含义 |
|---|---:|---|
| 没抄 | 10 | 全从问题推，或只是把事实自然用进推理（含"合法用事实"，**不算照抄**） |
| 抄1句 | 8 | 1 句换词复述参考 |
| 抄2句 | 6 | 2 句 |
| 抄3句 | 4 | 3 句 |
| 抄4句 | 2 | 4 句 |
| 完全照抄 | 0 | 整段几乎每句都在逐条复述参考 |

**每条 think 的真实档位是客观事实标准**，由 step160 从人工标签算出、留存（`true_level`/`level_idx`/`anchor`，端到端可追溯）。

**为什么这框架对**：模型一个题采 8 个候选，客观上分散在这些档位上；RL 要的是"把更脏的往更干净的推"的梯度。**如果 Kimi 在某段分不开（曲线压平），那段的真实质量差它就捕捉不到 → 即使模型真在进步，也喂不出梯度 = 强化学习到顶。**

## 3. 两个实验（一次采集、读两遍）

每条 think 整段让 Kimi 打一个 **0-10 干净分**（10=完全没换词复述照抄，0=整段都是照抄），**同一条独立打 16 遍**，一次采完。然后：

### 实验一 · 分数跳不跳 → 该打几遍（K）
- 单遍打分的典型标准差 σ；取平均后噪声按 σ/√k 缩。
- **K = 噪声压到 < 半个档距（1.0 分）所需的遍数**——低于这个，相邻档位会被噪声糊在一起。

### 实验二 · 分辨率曲线 → 饱和拐点 = RL 天花板【核心】
- 画 **Kimi 平均干净分 随 真实档位** 的曲线（理想：高度跟着锚点，10→8→6→4→2→0）。
- **① 单调性**：客观锚点↑ → Kimi 分↑ 的 Spearman（越接近 1 越好）。
- **② 相邻档位分不分得开**：相邻两档 Kimi 均值差 ≥1 且 1 倍标准差带不重叠 = 分得开。
- **③ 找压平区**：连续"分不开"的相邻档位 = 曲线趋于直线的那一段 = Kimi 没有分辨力的区间。
- **判读**：
  - 全部相邻档位分得开 → 无分辨率盲区，全程有可用 RL 梯度；
  - 在某段（如"抄4句~完全照抄"）压平 → **Kimi 在这段分不开，RL 在此到顶（天花板）**；
  - 单独盯 **没抄↔抄1句** 拐点：分不开 = 把"只用了事实"的干净 think 也压低分 = **把合法用事实误判成照抄（阶段7 的病）**。

> **必须分清的坑**：若模型采的 8 个候选"真的"都在同一档（没产出差），那是**模型没 headroom**（阶段5 的病），不是 Kimi 的上限；Kimi 的上限是"候选真有差、它却读不出"（实验二测的就是这个）。
> **读"压平"要看分布不看均值**：相邻档位均值接近、且标准差带重叠，才算真分不开（同档不同样本本身有散布）。若整条曲线都饱和，兜底改"同题两两比较"。

## 4. 四类标签怎么标（含例子）

标定数据 `data/judgecal_sentences.jsonl` 人工逐句标 4 类（verbatim/reworded/legit_use/original），step160 据此算出每条 think 的真实档位。以"增值税小规模免税"为例（参考含 a：月销售额未超10万免征；b：差额征税按差额后销售额判断）：

- **verbatim（原样照抄）**：「纳税人转让旧房…可按发票所载金额…每年加计5%计算。」
- **reworded（换词复述，Kimi 该抓）**：「小规模纳税人一个月卖不到10万，就能免增值税。」← a 的复述。
- **legit_use（合法用事实，不该当照抄）**：「本月9万，单看没过10万这条线。」← 用"10万"推理，不是复述 a。
- **original（纯原创）**：「这题要判断本月到底免不免，得先看销售额卡在哪条线上。」

一条 think 的真实档位 = 它含几句 copy（verbatim+reworded）；整段全是 copy = 完全照抄。

当前种子集：**78 条 think（13 个参考资料 × 6 档，每档 13 条）**，由 `code/data/build_judgecal_dataset.py` 受控生成——**每个参考拆 4 条款，每条款只写一次"换词复述"、跨档复用，所以改写难度跨档恒定，唯一变量是"复述了几条款"**；非照抄填充用原创+合法用事实，每条 think 长度拉平在 5 句(完全照抄 4 句)，不让长度成线索。**样本量加大到 13/档，摊平构造者(我)的偏差。**

**构造纪律——"换词复述"的力度由真规则客观标定**（`code/data/check_judgecal_vs_rule.py`，用 `reward_v3.masked_copy`=遮掉法定术语/数字后的字面重合）：
- 原样条款 masked_copy ≈ 1.0（规则该抓到）；
- **换词复述 masked_copy 实测 0.29，已和"合法用事实"的 0.26 同量级、远低于规则惩罚线 0.40**——即字面规则把"复述"和"正常用事实"都判成干净、分不开，**正是"介于规则无法可靠识别"，也证明非 Kimi 不可**；
- **整段 think 各档 masked_copy 全 < 0.40（完全照抄仅 0.15）**——字面规则对换词复述全盲，连"整段都是复述"都当干净；
- 残留约 6 个唯一句（烟酒枚举、"四级超率累进税率""高新技术企业"等术语/枚举密集到没连接结构可改的）masked_copy 仍 >0.40——这些**术语本身即字面铁证、规则抓得到，属"规则够用"的少数，不是 Kimi 专属**，保留不影响实验。

## 5. 怎么跑 / 产物

```bash
bash scripts/run_judgecal.sh        # 页面1（需 DASHSCOPE_API_KEY；离线零 GPU）
bash scripts/monitor_judgecal.sh    # 页面2
```
链路：`step160`（校验装配+算档位锚点）→ `step161`（每条 think 打 0-10 分 × 16 遍）→ `step162`（读两遍出报告）。Kimi 调用 ≈ 78×16 = **1248 次**。
产物 `output/judgecal/main/`：
- `160_judgecal_items.jsonl`：带 `true_level`/`anchor` 客观真值的规范化数据；
- `161_sentence_judges.jsonl`：每条 think × 16 遍的 0-10 分（**唯一花 Kimi 钱，断点续跑会跳过**）；
- `162_judgecal_report.md` / `_decision.json`：实验一二结论（σ→K / 分辨率曲线 / 饱和拐点 / 没抄↔抄1句拐点 / 离群清单）。

冒烟：`JUDGECAL_LIMIT=4 bash scripts/run_judgecal.sh`。
可调：`JUDGECAL_KMAX`(16)、`JUDGECAL_WORKERS`(3)、`JUDGECAL_SEP_MIN`(1.0=判"分得开"的最小分差)。

## 6. 决策树（这模块逼着做决定）

- **全 6 档分得开 + 没抄↔抄1句分得开** → 最好。下一模块：用 Kimi **离线标 DPO 对**（答案做硬门、对子只比 think、v5 answer-lock），可选细粒度对子。
- **中高档分得开，但脏端（如 抄4↔完全）压平** → 仍可训，**对子拉到压平区之外的大间距**（阶段8 说 79% 的题有全干净候选，够用）。
- **没抄↔抄1句就分不开** → Kimi 把合法用事实当照抄（阶段7 复发）→ 先改判官提示词/加"先指认参考出处"，再标定一轮。
- **整条曲线都饱和** → 绝对打分不行 → 改"同题两两比较"判法，或换更强裁判。
- **实验一 σ 太大、16 遍仍不够** → 加大 `JUDGECAL_KMAX` 重采。

---

## 7. 怎么扩数据集

往 `data/judgecal_sentences.jsonl` 加行，每行一条，逐句标 4 类；step160 自动算真实档位并严格校验。要补哪档就构造对应 copy 句数的 think（全 copy 句 = 完全照抄）。reworded/legit_use 这两类边界最关键，多构造、各档 ≥5 条曲线才稳。

> 注意：构造样本是"定点替换"式照抄，和真实弥散照抄不完全一样。结论 GO 后，应再抽几十条**真实模型采样**让人核对一次，确认这里测出的分辨率迁得过去。
