# 99 · 方向裁决：放弃自采样 RL 主线，转「受控重写 → SFT 自蒸馏」（五角色评审全票结论）

> 评审方式：项目目标/业务、Kimi 裁判能力、RL 可行性、数据构造、风险交付五个角色独立审查并强制表态。
> **表态结果 5/5：换方案但保留最低成本 RL 验证。** 无人支持继续或修补当前 RL。
> 本文档是给 Codex 的执行蓝本；统计与代码证据均已实算（来源 110-113 号产物 + judge_common.py + 历史复盘 98 号）。

---

## 0. 一句话结论

**放弃"自采样池 + DPO/GRPO"的 RL 主线（已被三轮受控实验在新旧两套口径下证伪），主交付转为「Kimi 受控重写 → 门控 → SFT 自蒸馏」（方案 A）；RL 仅保留 A 成功后 1 天时间盒的构造对 DPO 验证（方案 B）；GRPO 永久出局。**

## 1. 为什么（证据链，每条都有实验编号）

1. **RL 三连零，机制性而非执行性**：corrected-v1 DPO/GRPO×3（Δh −0.007~−0.010）、v2 mini DPO（Δh −0.009）、v3.1 新口径终审（Δtrace_free=+0.0005，CI[−0.012,+0.013]）。KL 锚定的偏好学习只能在 πref 已有模式间重新加权；best-of-8 复判 0.701≈base 0.697（16,120 个自采样）证明**结构性干净的 think 在策略自身采样分布里质量≈0**——这是 proposal distribution 的几何缺陷，调参没有出路。
2. **14B 全史是阳性对照**：Kimi改写→SFT +0.57（0.21→0.78）、DPO +0.056、GRPO +0.01——收益随"依赖自采样的程度"单调衰减。全项目唯一的大杠杆从来是"构造性注入分布"，方案 A 打回这个杠杆。
3. **新目标三件套（去痕迹/守答案/守依据）正确且首次可测**：新裁判噪声减半（trace_sd 0.056-0.060 vs 旧 0.117；r12 0.634-0.667 vs 0.434），E2 机械删痕迹 Δtrace_free=+0.038 CI[+0.005,+0.074] 且守护项不崩——尺子修好了。旧 humanness 被 E2 rewrite ceiling（0.740）实验性杀死，正式废弃。
4. **效应量这次算得过账**：base 带痕迹题 87/224（38.8%），若重写 SFT 推到 ~0.85，期望整集 +0.07；SFT 一般回收老师增量 50-70% → 预期 +0.07~+0.10，高于 +0.05 验收线、高于 MDE 0.013-0.018 一个量级。**与 v2（设计效应 0.006 < MDE 0.022）正好相反——这次开工前算清了能读出来。**
5. mini DPO 不伤准确率已被赦免（McNemar b=18 c=15 chi2=0.121），RL 主线可以干净收档，叙事是"三轮受控实验证明信号不在自采样分布内"，不是项目失败。

## 2. 目标与裁判的正式裁决

**目标**：采纳三件套，补四个护栏——①think-answer 结论一致性判项（现 grounded 模板不查 think 推 X 而 answer 说 Y）；②长度带宽（裁判和产品都对长度敏感）；③口癖/多样性监控（防"蒸出另一种机器腔"）；④**前置一行字：书面确认 think 是否展示给最终用户、是否流式**——全工作区没有任何文档记载，它决定全部预算的正当性（不展示→直接隐藏 think 收工）。

**Kimi 能力分项**：
| 任务 | 裁决 | 条件 |
|---|---|---|
| 判 RAG 痕迹 | 有条件能 | 聚合配对读数可用（MDE≈0.013）；单样本/逐类型不行（verbatim/policy_source k=3 非全票率 26-27%）→ 必须并联确定性检测器（正则 trace_hits + copy_ratio） |
| 判 answer-gold 一致 | 能 | 修掉 aggregate_judges 假多数票 bug 后，k=3 多数票+McNemar 作验收门；**A2 阶段不用它**——answer 逐字保留用代码 byte-equality 断言 |
| 判 grounded | 未验证 | 69% 判分聚在 0.9-1.0，可能是印象分；先过扰动测试（40 题×3 变体：改数字/换口径/加无依据事实，~240 调用；数字扰动 ≥80% 掉分才算过）才可作闸；并联双向数字守恒检查 |
| 判"像人" | **不能，正式退役** | E2 ceiling 0.740 + r=0.434 + 0.85 封顶 + pearson(h,痕迹)=−0.804 |
| 成对判分 | 弃作硬门 | short_win=23.3% 长度偏置 + 2% 位置偏置；只留作诊断 |

**裁判侧三个必修 bug**（A0 前完成）：①`aggregate_judges` 的多数票实为 round(mean(rank))，[correct,correct,incorrect] 会输出无人投的 partial（judge_common.py:229,238）→ 改真 mode、平票取保守；②parse 兜底把缺字段写 0.0（:130-131），一条 0 注入 60 样本均值即拉偏 ~0.013 ≈ 整个 MDE → 改"缺字段=作废重试"；③k=3 在 temp=0.0 下是伪重复（:169）→ judge temp 0.3 或轮换字段顺序变体；reference 截断 3000→6000（224 题 user_prompt p90=3982，现状两位数百分比样本 verbatim/grounded 全盲）。

## 3. 方案 A（主交付）：定向 de-RAG 重写 + SFT 自蒸馏

**A-1（零成本前置）**：组长/产品书面确认 think 展示方式与流式要求。

**A0（前置 GO 门，~540 调用）**：v3 已有 50 条受控重写用新口径 k=3 重判（它们全是评测集 query，**只作测量、禁止入训**）。GO 门按带痕迹子集分层：重写后 trace_free≥0.85 ∧ 多数票痕迹消失率 ≥70% ∧ Δgrounded≥−0.02 ∧ 确定性正则痕迹=0（混合均值门作废——抽样里 ~6 成本来就干净，基线 0.846，0.85 混合门白送）。同批跑 grounded 扰动测试。0.80-0.85 之间只许修一次重写 prompt 重测，不许降线。

**A1（数据源）**：**重推 greedy**（部署解码参数 max_new=1536），不用 60_dpo_rollout 的 temp0.9 候选（温度与截断双错位）。筛选：本地确定性预筛先行（trace_hits>0 ∨ copy_ratio≥0.30 直接进重写桶，零调用），Kimi 只兜灰区；**源样本 answer 门：k=2 acc 多数 correct 才入队**（堵 ~18% 错误答案蒸回模型——A 最大的隐性自伤点），错答案题不重写也不进 replay。

**A2（重写与门控）**：重写 prompt 以 step06 为底（保留治根条款"依据/数字/结论完全来自参考资料"与"只输出推理过程不输出答案"），新增两条："不要机械罗列文件号/政策名清单"（policy_source=61 主导桶）、"不要大段照搬参考原文，用自己的话转述"（verbatim_copy=52）。门控 = LLM 门（k=2 **两判 min** trace_free≥0.85 ∧ min grounded≥0.85）∧ 确定性闸（trace_hits=0 ∧ copy_ratio≤0.30 ∧ introduced_nums=∅ ∧ **missing_nums=∅**——step06 现有闸只查新增数字不查丢失，冷启动 grounding 事故的根因恰是删掉关键限制）∧ answer byte-equality 代码断言 ∧ 长度带宽 [0.6,1.3] ∧ 口癖前缀闸（开头 8 字前缀占比>5% 重写重试）。先 100 条小批验证吞吐与通过率再放全量过夜。

**A3（训练）**：fresh LoRA r16/α32/dropout0.05 全线性层，底座=冻结 RFT merged base（回滚=摘 adapter）；lr 5e-5~1e-4（与已验证的冷启动 SFT 同档）、epoch≤2、warmup 0.05；训练集=过门重写（目标 ≥500 条，按 87/224 痕迹率 ×2015×82% acc 门 ×~70% 过门率 ≈450-580 条）+ ~30% replay（**取 base 自己的 greedy 干净输出**，过 acc=correct 与 k≥2 干净复判，不用 Kimi 种子）；每 epoch 224 题 acc 快检，McNemar 显著退化即停回滚；训前 50 题格式 smoke（</think> 标签闭合率）。

**A4（验收，v3.1 框架）**：统计门=paired Δtrace_free 95%CI 下界>0 ∧ ≥+0.05；分层门=带痕迹子集（n=87）Δ≥+0.10 ∧ 干净子集 Δ≥−0.02；**痕迹计数硬指标**（组长可抽查）：verbatim_copy 52→≤30、policy_source 61→≤35、explicit_ref+ref_enumeration 题占比 9%→≤2%；守护=McNemar 中性 ∧ Δgrounded CI 下界 ≥−0.02 ∧ c+p%≥90%；口癖=任意 8-gram 出现于 >10% think 即 FLAG、distinct-2 降幅 ≤10%；业务口径=20-30 题非项目成员盲测"哪个更不像在念资料"胜率 ≥70%。门控判分与验收判分强制不同 run_id。

**A5（兜底，降级定位）**：本地正则安全网常驻（只删 explicit_ref/ref_enumeration 显式标记 + 清理 pass，流式安全，只在 think 段内编辑、保留标签结构）；上限已实测 +0.038，**不是终局，不得在叙事中顶替 A**；serving 不调公网 Kimi（内网合规+流式双杀）。

**预算与工期**：~8k Kimi 调用（一夜级）+ GPU（greedy 重推 1-2h + SFT 1-2h）；名义 3 个工作日，**7 个日历日硬上限**（含一次灰区补救），超期即交付 A5+测量报告收工。

**若 A 也读不出 +0.05，按序怀疑**：①迁移失败（对比输出与 target 的痕迹计数、train loss）；②裁判 0.85 附近饱和（对 A 输出再跑一次 surgery：计数降分数不动→修锚点）；③脏 replay/误路由（k=3 重判 replay）。升级一次（全池重写+LoRA r32 或低 lr 全参）后仍无效→接受"剩余 headroom 裁判不可达"，交付现状。

## 4. 方案 B（最低成本 RL 验证，条件触发）

**仅在 A 通过 A4 后**，1 个日历日时间盒，单次读数不迭代：从 **A 的 checkpoint** 续训 DPO；rejected=A 模型训后自采样的**残留痕迹输出（此时才是 on-policy）**，chosen=对应过门重写 + ~15% 锚定对（chosen=模型自己的干净 greedy 输出）；300-500 对、beta 0.1、lr 5e-6、rpo_alpha=1.0、epoch≤2、pair 级长度带宽 |ratio−1|≤0.3。**预注册判定**：相对 A checkpoint（不是 RFT base）paired Δtrace_free CI>0 ∧ 痕迹计数更低 ∧ McNemar 中性 ∧ 80 题池外 holdout 增益不缩水——四条不全则如实写"构造对 DPO 相对蒸馏无增量"收口。叙事必须改为"构造性偏好对 DPO"，不得再称"纯自我改进 RL"（技术方案.md:20-21 的原承诺需在终报主动改口径：外部老师只供叙事风格重写，事实与答案锚定 V1 与参考资料）。

## 5. 立即执行清单（按优先级）

1. **问组长/产品两个问题**（think 是否展示？是否流式？）——零成本，决定一切预算正当性。
2. 修 judge_common 三 bug + reference 截断 6000（半天，影响所有后续读数）。
3. A0：50 条重写新口径分层重判 + grounded 扰动测试（~540 调用，<1h）→ GO 才往下。
4. A1：greedy 重推 2015 题 + 确定性预筛 + acc 门（GPU 1-2h + ~2k 调用）。
5. A2：100 条小批验证 → 全量重写+门控过夜（~5.5k 调用）。
6. A3+A4：SFT 1-2h + 验收 ~1.3k 调用。
7. （条件触发）B 的 1 天时间盒。
8. 终报：crosswalk 一页（旧 h=0.697 ↔ 新口径 0.775/0.867/0.825，旧 h 出现一次后永久封存）+ 叙事框架："指标升级而非目标更换；RL 不是被放弃，是被三轮受控实验证明不适配该信号分布；交付物不变，方法换成测量验证过的自蒸馏。"

## 6. 不要再做的实验（止损清单）

1. 任何自采样池上的 DPO/GRPO——三轮零收益 + 0.701≈0.697 已封死，机制问题不是超参问题。
2. **GRPO 永久出局**：本地 reward 只覆盖正则+copy_ratio 可算的切片（= sed 能做的 +0.038），结构性痕迹无法本地计算；在线 LLM reward 吞吐（19 次/min）与噪声（组内真实离散度≈0 << 裁判 sd）双不达标。
3. 旧 humanness 口径的任何优化、汇报、决策（ceiling 0.740 已死）。
4. 未校准成对判分作硬门（长度偏置 23.3%）。
5. k=1 单判做任何 GO/NO-GO、筛选门或 margin。
6. 降门续跑 / WARN 旁路 / fail-open（80%→70%→59.2% 放行与 headroom 门是本项目最大流程事故）；A0/A2/A4 全部 raise 硬停。
7. 在同一份判分上既选样又汇报（95b 教训）。
8. A4 灰区第二次补救；7 日历日超期即收工交付。
9. 任何含"与 gold 字符相似度"的正向 reward 项（照抄泵）。
10. v3 的 50 条重写或任何 224 评测集 query 进训练数据（代码 assert pool∩eval=∅，违反硬停）。

## 7. 简历/面试可保留的资产（风险/交付 agent 整理）

① reward hacking 从现象（verbatim_copy 60→63）追到机制（与 gold 的字符相似度泵）的定位过程；② "先建尺子再优化"：σ=0.117/r=0.43 的裁判读不出 MDE 0.022 → 设计效应开训前即注定失败的复盘；③ 零训练证伪探针（placebo surgery / rewrite ceiling / pairwise 偏置检测），用一天 API 成本避免一轮 GPU 训练；④ RL→编辑式自蒸馏的方法论转身及完整依据链；⑤ 门被旁路的流程事故与硬停修复。"严谨测出四轮无效并定位原因"比"+0.01 的 RL 收益"在面试中更有区分度。
