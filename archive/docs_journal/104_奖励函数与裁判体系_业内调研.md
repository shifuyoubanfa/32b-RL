# 32B 去 RAG 强化学习：奖励函数与裁判体系业内调研

> 调研日期：2026-06-12  
> 目标：在税务问答答案正确性不退化的前提下，让 `think` 减少机械 RAG 痕迹，并让 DPO、GRPO 的收益可以被可靠证明。  
> 方法：一手论文/官方资料检索 + 当前代码审计 + 三条独立 Agent 调研 + 交叉审查。

## 1. 执行结论

业内没有一个成熟的单一奖励函数，能够在开放域、事实密集的税务问答中，同时可靠优化：

1. 去除机械 RAG 痕迹；
2. 保持答案正确、事实完整；
3. 不被模型钻奖励漏洞。

论文和公开工程实践较一致地支持以下路线：

> **把正确性、关键事实保持建模为不可被风格收益抵消的约束；只在事实合格候选之间优化“去 RAG”目标。**

这意味着本项目不应继续把模糊的 `humanness`、单一 `trace_total` 或 Kimi 二分类结果直接当作主奖励。推荐的目标是：

```text
在正确性与关键事实非劣约束成立的可行域内，
优化明确、可审计、相互独立的去 RAG 指标。
```

Kimi 在当前项目中最合适的角色是：

- 离线弱标签器；
- 原子事实拆解与事实风险发现器；
- 规则无法判断时的争议样本复核者；
- 独立最终审计裁判之一。

当前 Kimi 校准结果 `balanced_accuracy=0.6333`、坏样本误放率 `43.33%`，不足以担任：

- DPO 数据的唯一硬门禁；
- 在线 GRPO 的唯一奖励；
- 最终验收的唯一裁判。

## 2. 业内如何处理类似问题

### 2.1 可验证任务：优先使用规则奖励

DeepSeek-R1 在数学、代码等可验证任务中主要使用正确性规则奖励和格式奖励，并明确避免依赖通用神经过程奖励模型。原因是中间推理过程难以可靠定义，奖励模型容易被投机。

这对本项目的含义不是“全部改成规则”，而是：

- 格式、显式 RAG 痕迹、长段复制、数值计算等可验证部分尽量规则化；
- 税务事实语义等不可完全规则化部分交给离线裁判、检索验证和人工校准；
- 不让一个模糊 LLM 总分支配在线 RL。

### 2.2 多目标任务：拆分奖励与约束

Safe RLHF、Constrained RLHF、MODPO 等研究的共同方向是：复杂目标不能简单压成一个总分。一个目标的收益不应任意抵消另一个关键目标的损失。

本项目的词典序应是：

1. 输出格式有效，不退化；
2. 最终答案与关键税务事实正确；
3. 必要条件、例外与结论覆盖完整；
4. 减少机械 RAG 痕迹和不必要复制；
5. 其他表达风格优化。

其中第 2、3 项是约束，第 4 项才是主要优化目标。

### 2.3 事实密集问答：原子事实而非整体印象

FActScore、SAFE、ALCE 等事实性评测将长答案拆成原子事实，再判断事实是否被可靠证据支持，并同时关注覆盖度。原因是只优化事实精度会出现捷径：模型通过少说、重复简单事实获得高分，却没有真正回答问题。

税务场景需要重点保护：

- 纳税主体；
- 税种与适用期间；
- 税率、阈值、计税基础；
- 前置条件与例外条件；
- 数值计算；
- 最终结论。

“法规名称或事实落点”可以保留；需要消除的是“根据提供的资料一”“参考文档显示”等系统/RAG 痕迹。两者必须分开。

### 2.4 LLM 裁判：先校准，再使用，并允许弃权

LLM-as-a-Judge 研究反复发现位置偏差、长度偏差、自偏好、领域能力不足和对抗样本脆弱性。可靠使用方式通常包括：

1. Pointwise 分项检查每个答案；
2. Pairwise 比较 A/B；
3. 交换为 B/A 再评一次；
4. 两次冲突则 `ABSTAIN`；
5. 对争议样本使用异构模型或人工复核；
6. 在固定人工金标集上报告 sensitivity、specificity、balanced accuracy、Cohen's kappa 和顺序一致率。

同一个 Kimi、同一提示词重复多次，只能缓解部分随机噪声，不能修复共同的系统偏差。

### 2.5 奖励过优化：代理分数上涨不等于质量上涨

奖励模型过优化、DPO 过优化和长度投机研究均显示：模型会主动寻找代理指标漏洞。常见现象包括：

- 显式引用减少，但逐句复制增加；
- 回答变短，事实错误看似减少，但信息覆盖也下降；
- 风格更像裁判偏好的写法，但内容没有改善；
- chosen 和 rejected 的概率同时下降；
- 训练奖励上涨，独立评测下降。

因此必须使用独立于训练奖励的冻结审计集，并逐项检查指标迁移。

## 3. 当前项目为何会接受退化结果

### 3.1 在线 GRPO 奖励未直接保护答案正确性

当前 [`reward_v3.py`](code/pipeline/reward_v3.py) 的 `derag_reward()` 先用弱事实门禁拦截部分输出，但一旦通过，正奖励主要由痕迹分决定。答案正确性没有持续进入正奖励排序。

风险：

- 一个“事实词重叠尚可但结论错误”的答案可能得到较高奖励；
- GRPO 会学习如何穿过弱门禁，而不是稳定保持答案正确；
- 风格收益可能掩盖事实退化。

### 3.2 DPO pair 混入了答案质量差异

当前 [`step124_build_dpo_pairs_v4.py`](code/pipeline/step124_build_dpo_pairs_v4.py) 允许：

```text
chosen.answer_score >= 0.55
rejected.answer_score >= 0.45
chosen.fact_recall 最多比 rejected 低 0.05
```

这不能保证 chosen/rejected 只在 RAG 痕迹上不同。DPO 可能同时学习到：

- 更干净的风格；
- 不同答案内容；
- 不同事实覆盖；
- 不同长度与表达偏好。

最终无法归因模型到底学到了什么。

### 3.3 当前验收门禁允许用质量损失换痕迹下降

当前 [`run_derag_v4.py`](code/run_derag_v4.py) 的候选比较逻辑允许：

```text
answer_score 在容忍线内下降
且 trace_total 至少下降一点
```

这次 DPO-a 的实际变化是：

| 指标 | S1 | DPO-a | 变化 |
|---|---:|---:|---:|
| clean rate | 0.2411 | 0.2366 | -0.0045 |
| trace total | 127 | 120 | -7 |
| explicit_ref | 25 | 15 | -10 |
| verbatim_copy | 26 | 30 | +4 |
| answer score | 0.7034 | 0.6922 | -0.0112 |
| fact recall | 0.7779 | 0.7696 | -0.0083 |
| 配对净改善 | - | 14 improved / 13 worsened | +1 |

这不是被证明的提升，而是典型的：

- 指标迁移：`explicit_ref` 下降，`verbatim_copy` 上升；
- 质量退化：answer、fact、clean 同时下降；
- 统计不确定：配对净改善只有 `+1`。

当前 `ok=true` 只表示通过了旧门禁，不能解释为模型变好。

## 4. 推荐奖励函数

### 4.1 不再使用单一加权总分

不推荐：

```text
reward = correctness + fact + humanness + derag
```

因为风格收益可能补偿错误答案。

推荐使用分层奖励：

```text
若格式退化：
    reward = -2

若高置信答案错误、关键事实错误或遗漏会改变结论的条件：
    reward = -1

若正确性/事实性不确定：
    不参与该组梯度更新，或给予统一 reward 使组内 advantage 为 0

若正确性与事实约束通过：
    reward = 1
             + bounded_derag_reward
             + bounded_completeness_reward
             - uncertainty_penalty
```

必须保证：

```text
任何事实合格候选的最低奖励
>
任何事实不合格候选的最高奖励
```

### 4.2 去 RAG 奖励使用多维保守聚合

不要只用 `trace_total` 求和，否则一种痕迹减少、另一种增加仍可能得高分。

建议独立记录：

- `explicit_system_reference`：根据提供资料、参考文档等；
- `reference_enumeration`：资料一、资料二；
- `verbatim_copy`：非必要长连续复制；
- `source_order_mirroring`：按资料顺序机械复述；
- `template_repetition`：模板化过渡语重复；
- `policy_source_false_positive`：正常法规引用误伤率。

奖励采用最差维度或尾部风险聚合：

```text
S_derag = 1 - max(
    normalized_explicit_system_reference,
    normalized_reference_enumeration,
    normalized_verbatim_copy,
    normalized_source_order_mirroring,
    normalized_template_repetition
)
```

这样模型不能通过把一种痕迹迁移成另一种痕迹获利。

### 4.3 GRPO 组内样本处理

对每个问题采样 `K` 个候选：

1. 执行格式与确定性规则检查；
2. 执行事实/正确性高置信检查；
3. 不确定候选标记为 `ABSTAIN`；
4. 只在事实合格候选之间计算去 RAG 排序；
5. 若组内合格候选少于 2 个，扩大 K 或跳过该题；
6. 若合格候选风格分没有足够方差，跳过该题；
7. 奖励有界，保留 KL 约束和早停。

关键点：全错组不能因为“有的答案更干净”而产生正向风格梯度。

## 5. 推荐 DPO 数据设计

### 5.1 偏好顺序

每个税务问题构造：

| 类型 | 答案/事实正确 | 无机械 RAG 痕迹 |
|---|---:|---:|
| A | 是 | 是 |
| B | 是 | 否 |
| C | 否 | 是 |
| D | 否 | 否 |

偏好至少满足：

```text
A > B
A > C
B > C
```

`B > C` 明确告诉模型：正确但机械的答案，仍优于自然但错误的答案。

### 5.2 高置信 pair 标准

用于主 DPO 训练的 pair 应尽可能满足：

- 同一问题；
- 最终结论相同；
- 原子事实集合等价；
- 关键条件与例外等价；
- 长度大致匹配；
- 主要差异是机械 RAG 痕迹；
- Kimi 双序评审一致；
- 规则与 Kimi 不冲突；
- 不确定样本直接弃权。

最优 pair 是“最小编辑反事实”：只删除/改写 RAG 痕迹，不改动答案语义。

对困难 pair 可使用置信度加权 DPO；主训练集不应混入低置信标签。

## 6. Kimi 的合理职责

### 6.1 应做

- 将答案拆为原子事实；
- 对照权威资料判断支持关系；
- 标出可能改变结论的遗漏与错误；
- 对规则无法判断的风格边界做复核；
- 对 DPO pair 做 A/B、B/A 双序审查；
- 对独立审计集提供一个裁判视角。

### 6.2 不应做

- 使用抽象 `humanness` 一次判断全部目标；
- 被迫对每个样本做 PASS/FAIL；
- 作为在线 GRPO 唯一奖励；
- 同时担任教师、唯一裁判和最终验收者；
- 在未通过人工校准时作为硬门禁。

### 6.3 输出协议

```json
{
  "derag_verdict": "A|B|TIE|ABSTAIN",
  "correctness": "PASS|FAIL|UNCERTAIN",
  "fact_risks": [],
  "evidence_spans": [],
  "confidence": 0.0
}
```

无具体证据、低置信或双序冲突时必须弃权。

## 7. 如何证明模型真的提升

### 7.1 先测评测噪声

在冻结基线模型上重复推理和重复裁判，分别测量：

- 模型采样方差；
- Kimi 随机方差；
- Kimi A/B 顺序偏差；
- 规则误伤率；
- 人工标注分歧。

正确性容忍线 `epsilon_answer`、事实容忍线 `epsilon_fact` 应由这些噪声和业务容忍度共同确定，不能直接拍定 `0.02`。

### 7.2 验收必须同时满足

在冻结、配对、独立评测集上：

1. **正确性非劣效成立**  
   `LCB95(delta_answer) >= -epsilon_answer`

2. **关键事实非劣效成立**  
   `LCB95(delta_fact) >= -epsilon_fact`

3. **去 RAG 优效成立**  
   `LCB95(delta_derag) >= minimum_effect`

4. **没有指标迁移**  
   每个主要痕迹分量均不显著退化，或总体最差项改善。

5. **独立裁判与人工抽检支持结论**

统计方法：

- 二元配对结果：exact McNemar；
- 连续指标、胜率、均值差：按题目 paired bootstrap；
- 多次裁判：报告一致率与置信区间；
- 多版本尝试：控制反复试验导致的假阳性。

“下降没超过容忍线”只能证明没有检测到明显退化，不能证明优化成功；去 RAG 指标必须达到预注册的最小有效改善量。

## 8. 最小可行落地路线

### 交叉审查后的关键修正

三条调研线彼此攻击后，确认原始建议还需增加以下限制：

1. **所谓硬门禁必须是高精度、低覆盖率门禁。**  
   开放税务题无法全部自动验证。候选必须分为：

   ```text
   VERIFIED_VALID    有可靠证据证明合格
   VERIFIED_INVALID  有可靠证据证明错误
   UNKNOWN           无法可靠判断
   ```

   `UNKNOWN` 不能当作通过或失败，应退出训练梯度并进入审计池。

2. **扩大 K 会同时放大极值选择和误放风险。**  
   若单候选坏样本误放率为 `f`，K 个候选中至少一个坏样本被误放的概率为：

   ```text
   1 - (1 - f)^K
   ```

   当前 Kimi 坏样本误放率为 `43.33%`，在 `K=16` 时几乎必然出现误放。因此，扩大 K 只能搭配高精度验证器，并且必须二次审计最终极值候选；不能靠重复 Kimi 判分支撑大 K。

3. **正确性平均分非劣还不够。**  
   税务场景需要额外约束“关键事实错误数不得增加”。严重错误不能被其他题目的微小改善抵消。

4. **建议为可训练题目建立 Fact Contract。**  
   对结论、required claims、critical slots、forbidden claims、allowed numbers 做结构化约束。当前 `answer_score`、`fact_recall`、`grounding_floor_ok` 只能保留为监控指标，不能单独证明事实正确。

5. **训练裁判、开发裁判、最终裁判必须隔离。**  
   反复在同一 224 题和同一裁判上挑选 DPO/GRPO 变体，会导致适应性过拟合和伪提升。

6. **最终评测必须使用原始完整题集。**  
   不能只报告通过事实门禁的子集；否则模型可通过制造更多 `UNKNOWN` 提高条件指标。应同时报告全量结果、VALID 子集结果及 UNKNOWN/拒绝率。

### 阶段 A：冻结审计协议

1. 冻结一套独立评测集与人工金标子集；
2. 冻结去 RAG 各分量定义；
3. 校准 Kimi，增加 `UNCERTAIN/ABSTAIN`；
4. 增加 exact McNemar 与 paired bootstrap；
5. 预注册非劣界限和最小有效改善量。

### 阶段 B：重建 DPO pair

1. 保留双方答案与事实均合格的 pair；
2. 优先最小编辑反事实 pair；
3. 执行 Kimi pointwise + pairwise + 双序审查；
4. 丢弃分歧、不确定、答案内容不同的 pair；
5. 加入 `B > C`，强化“正确性优先”。

### 阶段 C：改造 GRPO 奖励

1. 仅对有可靠证据的题目建立高精度事实门禁；
2. `UNKNOWN` 候选不产生风格梯度；
3. 合格候选内使用有界多维去 RAG 奖励；
4. 合格候选不足时，仅在验证器误放率足够低时扩大 K，否则跳过；
5. 设置 KL、早停、独立审计。

### 阶段 D：版本验收

只有正确性与事实非劣、去 RAG 显著优效、无指标迁移三者同时成立，才能进入下一阶段或宣称 RL 有效。

## 9. 对当前实验的判断

- 当前链路工程上已经跑通；
- 当前 Kimi 二分类门禁未达到硬裁判要求；
- 当前 DPO-a 的 `ok=true` 是旧门禁产物，不是已证明的质量提升；
- 当前训练可作为探索性实验继续观察，但最终结果必须按新的独立验收协议重判；
- 下一版首要工作不是继续调学习率，而是重构奖励可行域、DPO pair 和验收统计。
- 当前使用大 K 的策略只能用于扩大候选池，不能默认提高训练数据质量；候选进入训练前仍需高精度事实验证与极值审计。

## 10. 多 Agent 交叉审查记录

本次调研使用三条独立路线：

1. 约束/多目标 RLHF 与奖励过优化；
2. LLM 裁判偏差、校准与统计验收；
3. 去 RAG 风格、事实性保护与奖励投机。

第一轮三方独立得出的共同结论：

- 正确性不能被风格奖励补偿；
- `humanness` 过于抽象；
- Kimi 不应成为唯一在线奖励或最终裁判；
- 当前 DPO-a 不能证明提升。

第二轮交叉审查主动攻击原方案，得到的主要反对意见：

- 双序 Kimi 只能缓解位置偏差，不能证明事实判断正确；
- 同一 Kimi 多阶段调用不等于多个独立证据；
- 事实门禁若误放率高，扩大 K 会显著放大坏候选混入；
- 冲突弃权会形成选择偏差，复杂税务题可能被系统性排除；
- 多维最差项仍可能被噪声最大的维度控制；
- 只在门禁通过子集上评测会产生条件选择偏差；
- 反复在同一验证集选参数会破坏统计有效性。

三方最终共同接受的修正版是：

> **不追求覆盖所有税务题的“全知裁判”，而是在能够高置信确认答案没有变错的狭窄可训练区域内做 DPO/GRPO；对无法判断的样本保留 UNKNOWN 并进入审计。**
