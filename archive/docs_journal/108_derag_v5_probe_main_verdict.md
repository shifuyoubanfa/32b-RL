## 但 X 不可信：挑题口径 vs 自救口径错配（致命）

- s150 **挑病题**用 Kimi（看得见结构性念手册）→ 挑出 489。
- s153 **判"think 干净"**(X 的分子) 用**确定性规则** `real_trace`（只抓字面词 explicit_ref/ref_enumeration + copy≥0.40，排 policy_source）——**正是对结构性念手册全瞎的那把尺子**。
- s154 的 Y 同样用规则 `is_clean` 判，同病。

**我们上轮只修好了挑题(规则22→Kimi489)，没修自救判定。**

### 确定性证据（交叉表）
1. 489 病题里 **规则全瞎(det_trace=0) 占 479/489 = 98%**；Kimi 标结构痕迹的 488/489。
2. X 按"规则能否看见此病"拆开：规则全瞎的 479 道，自救率仍 0.848、clean/16 仍 13.5 = **和总体一样**。规则在它本就看不见病的题上照样判 13.5/16 干净 = 橡皮图章。
3. **自救失败的 73 题里，72 题栽在"答案漂移 V1"，只有 1 题栽在"think 不干净"。** → **clean 闸几乎从不卡人**，X≈0.851 本质测的是"≥1/16 答案不漂 V1"（RFT≈V1 故必然高），与"能不能想干净"基本无关。

### 5 裁判面板（18 例"规则判净"样本，独立判 [B] 是否真无结构性念手册）
- 单裁判 genuinely_clean_frac：0.28 / 0.722 / 0.222 / 0.50 / 0.444，**均值 0.434、中位 0.444**。
- 多数票(≥3/5)：真干净 10/18=0.556，同风格(规则瞎判) 8/18=0.444。
- **全票脏的 5/14/17 全是政策法规类长答**（40号公告/医疗票据办法/25号公告），verbatim_copy 主导、Kimi trace_free 偏高——正是规则盲区最深处。模型把"念手册腔"去掉显式壳子(根据参考资料/资料1说)、改口语过渡，但**逐条搬运/归纳材料的本质未变**；例 5/7/13 的 [B] 甚至比 [A] 搬得更密。
- 好消息：8/10/11 全票真干净——模型**确实会**用独立推理去 RAG（收敛场景、因果短推理、改判结论）。所以 headroom 真实存在，只是远不到 0.85，且现尺子既看不见也奖不到。

## 裁决

**不能照单全收 GO_RL。** X=0.851 的 think-clean 维度被规则系统性高估约一档（真值 0.44–0.56），且 clean 闸不卡人、对自救率是虚加项。真正可信的只有"答案不漂 V1"（本就高）。

**两件事必须先做，再谈训练：**
1. **用 Kimi/结构感知裁判重测 X**（离线、几乎不耗 GPU）：151 的 16×489 采样已在盘，先用确定性 in-support 过滤（可信），再对 in-support 的样本用 Kimi 判 think 干净，得到真 X。这是 1 天级、API 成本的事。
2. **GRPO 的痕迹 reward 项绝不能用这套字面正则**：它已在 13.5/16 饱和（无梯度）、且对结构性搬运全瞎，直接拿来当 reward = GRPO 立刻 Goodhart 成"改头换面的复述"。痕迹项要换成结构/语义级——Kimi 离线 shaping，或"think 对参考材料的逐条对齐/语义搬运比例"（注意：literal copy_ratio 中位仅 0.235 < 0.30 阈值，paraphrase 式搬运靠字面重叠抓不到，需语义对齐如 embedding/NLI）。

**一句话**：这是同一个"尺子是问题"第 N 次复现，但这次定位到位——决策不是"训不训"，而是"先花 1 天把 clean 测量换成语义尺子，让真 X 来定 GO/NO-GO"。
