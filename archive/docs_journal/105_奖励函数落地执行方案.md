# 105 奖励函数落地执行方案

> 依据：`104_奖励函数与裁判体系_业内调研.md`  
> 目标：继续保留 DPO + GRPO 路线，但把训练信号改成“正确性可行域内优化去 RAG”，避免模型用答案质量下降换取表面去痕迹。

## 一句话打法

不要再问模型或 Kimi：“这个回答是不是更像人？”

改成：

```text
这条答案是否高置信没有变错？
如果没有变错，它是否减少了明确机械 RAG 痕迹？
如果无法判断是否变错，它就不作为训练信号。
```

## 当前必须废弃或降级的东西

1. `humanness` 不再作为主奖励，只保留为观察指标。
2. `trace_total` 下降不能单独证明变好。
3. `answer_score` 平均分不下降不能单独证明正确性没坏。
4. Kimi 二分类不能作为硬门禁，目前坏样本误放率过高。
5. `compare_gate()` 中“answer 允许小降 + trace 少 1 就过”的逻辑不能再作为阶段成功标准。
6. 大 K 采样不能默认提高样本质量，必须先有高精度事实验证器。

## 推荐新增核心概念

### 1. Fact Contract

每道可训练题生成结构化事实契约：

```json
{
  "conclusion": "最终结论",
  "critical_slots": {
    "tax_type": "税种",
    "subject": "纳税主体",
    "rate": "税率/金额/期限",
    "condition": "适用条件",
    "exception": "例外条件"
  },
  "required_claims": [
    "必须保留的关键事实"
  ],
  "forbidden_claims": [
    "一旦出现就代表变错的事实"
  ],
  "allowed_numbers": [
    "允许出现的关键数字"
  ]
}
```

Fact Contract 不是为了覆盖全部税务题，而是为了在可验证样本上得到高精度训练信号。

### 2. 三态事实裁判

所有候选分为：

```text
VERIFIED_VALID    有证据证明答案与关键事实安全
VERIFIED_INVALID  有证据证明答案或关键事实错误
UNKNOWN           无法可靠判断
```

处理方式：

| 状态 | DPO | GRPO |
|---|---|---|
| VERIFIED_VALID | 可进入风格比较 | 可计算去 RAG 奖励 |
| VERIFIED_INVALID | 不做 chosen；可做硬负样本 | 固定负奖励 |
| UNKNOWN | 不进训练主信号 | mask/skip，不产生风格梯度 |

### 3. 去 RAG 多维向量

不要只看总数，至少拆成：

- 显式系统资料痕迹：如“根据提供的资料”“参考资料显示”；
- 资料枚举痕迹：资料一、资料二；
- 长段复制；
- 按资料顺序机械复述；
- 模板化重复；
- 正常法规引用误伤。

最终验收要看每一维，不能让 `explicit_ref` 降低掩盖 `verbatim_copy` 上升。

## 代码落点

### `pipeline/reward_v3.py`

新增函数：

```python
fact_contract_pass(answer, think, contract) -> dict
answer_lock_pass(answer, baseline_answer, contract) -> dict
reasoning_integrity_pass(think, contract) -> dict
derag_style_vector(think, references) -> dict
derag_style_score(style_vector) -> float
```

改造 `derag_reward()`：

```python
if format_or_degenerate:
    return -2.0

if fact_status == "VERIFIED_INVALID":
    return -1.0

if fact_status == "UNKNOWN":
    return 0.0  # 或由上游 mask/skip

if reasoning_integrity_fail:
    return -0.5

return 0.2 + 0.8 * bounded_derag_style_score
```

注意：`answer_score`、`fact_recall`、`grounding_floor_ok` 只做监控，不作为充分正确性证明。

### `pipeline/step124_build_dpo_pairs_v4.py`

重建 DPO pair 筛选：

```text
先筛双方事实三态：都必须 VERIFIED_VALID
再筛 final answer/critical slots 等价
再筛 required claims 覆盖接近
再筛长度差不过大
最后按去 RAG 向量差异选 chosen/rejected
```

禁止再使用：

```text
chosen.answer_score >= 0.55
rejected.answer_score >= 0.45
```

作为主筛选标准。

### `pipeline/step13_build_grpo_data.py`

每条 GRPO 数据附加：

```json
{
  "fact_contract": {},
  "baseline_answer": "",
  "baseline_answer_signature": "",
  "required_claim_ids": []
}
```

### `swift/grpo_reward_plugin.py`

新增奖励函数，例如：

```text
derag_v5_lexicographic
```

必须满足：

```text
错误答案永远低于事实合格答案；
UNKNOWN 不制造风格梯度；
去 RAG 分数有界；
每个样本输出 reward breakdown，便于审计。
```

### `run_derag_v4.py`

替换 `compare_gate()`：

```text
1. 关键事实错误数不增加；
2. 正确性 paired-bootstrap 非劣效；
3. 去 RAG 主指标显著优效；
4. 各痕迹维度无指标迁移；
5. UNKNOWN/拒绝率不能恶化到不可接受；
6. 全部满足才进入下一阶段。
```

当前 DPO-a 这种 “trace 少一点、answer/fact/clean 都下降” 应判为 FAIL 或 exploratory-only。

## Kimi 的新职责

Kimi 不做在线 GRPO reward。

Kimi 做：

1. Fact Contract 生成与复核；
2. 原子事实风险分析；
3. DPO pair 的灰区双序审查；
4. 最终审计之一；
5. 对规则/Kimi/人工分歧样本写解释。

Kimi 输出必须允许：

```text
PASS / FAIL / UNKNOWN
```

并要求提供证据 span 或事实风险。没有证据就不能硬判。

## 下一轮实验建议

### Phase 0：离线重判当前结果

对当前 `20260612_151520` 这类结果先不急着继续宣称成功，而是重判：

- S1；
- DPO-a；
- 后续 GRPO 若有结果。

按 104/105 的新标准，判断哪些只是旧门禁通过，哪些真有证据变好。

### Phase 1：建立小而干净的 Fact Contract 集

先取 100-200 道题，优先选：

- 标准答案明确；
- 税率/主体/条件清晰；
- 参考资料能覆盖关键事实；
- 人工可快速抽查。

目标不是覆盖全量，而是获得高精度训练信号。

### Phase 2：重建高置信 DPO pair

只用：

```text
事实等价 + 风格显著不同
```

的 pair。数量少也可以，先追求干净。

### Phase 3：小规模 DPO

低学习率、少 epoch、chosen anchor，跑完后用冻结审计集判断：

- 答案是否非劣；
- 去 RAG 是否显著；
- 是否发生指标迁移。

### Phase 4：GRPO

仅在每题至少两个 `VERIFIED_VALID` 候选且风格有差异时训练。

如果 VALID 少：

- 先扩 K；
- 扩 K 仍少则跳过；
- 不允许 UNKNOWN 候选用风格分进入梯度。

## 成功标准

一次训练只有同时满足下面三条，才算有效：

```text
正确性非劣效成立；
关键事实错误不增加；
去 RAG 指标显著改善且无指标迁移。
```

如果只满足 “看起来更干净”，但 answer/fact/clean 下降，那不是成功，是奖励投机。
