# sandbox-14b/ — 前置 14B 沙盒实验（32B 立项的依据）

这是 32B 正式强化学习**立项之前**的可行性验证：在一个小模型上先证明"用 RL 能把税务大模型的
`<think>` 从 RAG 检索腔优化成像人的端到端推导、同时不掉准确率"。沙盒验证成功后，公司才把 32B（V1 本体）
交给我做正式 RL。

- **底座**：DeepSeek-R1-Distill-Qwen-14B（注意：原 `README.md` 标题模板里写的"7B"是勘误，实际是 14B；
  最初确实试过 7B，但 SFT 后 224 题里只 3 条会输出 `<think>`，被弃用——这是有记录的第一个弯路）。
- **三角色**：teacher=公司 V1 税务模型；student=上面的 14B；judge=跨厂家 Kimi（避免同系 self-preference 偏置）。
- **成果**：think humanness 0.212→**0.846**（≈4×）、RAG 显式痕迹清零、准确率仅 −0.03 → 命题被正向验证。
- **与 32B 的承接**：五阶段链（冷启动→RFT→DPO→GRPO）、本地 reward 选样 + Kimi 验收的分工、乘法门控奖励、
  非对称 KL、πref 锚上一阶段策略、"先证伪再烧卡"纪律——都被 32B 沿用；蒸馏 SFT 那步被推翻（32B 里 V1 就是本体）。

详细解读见仓库根 `../32B强化学习_实验报告.md` 的"前置沙盒验证（14B）"一章。
