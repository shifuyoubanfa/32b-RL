# 32B（公司 V1）推理人性化 RL — 启动手册

> 把 14B 沙盒验证过的方法（冷启动洗 think 风格 → RFT → DPO → GRPO；本地奖励选样 + Kimi 验收），
> 迁到公司 **V1（32B）本身**做自我改进 RL。目标：把 V1 的 think 从机器检索腔变自然，**绝不掉 V1 已有的准确率**。

---

## 0. 与 14B 沙盒最关键的差异：V1 既是"模型"又是"老师"

14B 沙盒里 V1 是老师、14B 是学生（先蒸馏、再 RL）。**32B 这里 V1 就是要改的模型本体**，所以：

- **跳过蒸馏 SFT（原 step03 / step04 的"蒸馏成学生"那步）**——V1 已经是训练好的模型，不需要再蒸馏进谁。
- LoRA 的 **base = V1 32B 本体**（`STUDENT_LOCAL_DIR` 指向 V1 权重目录）。
- 仍要跑 **V1 自产 think+answer（原 step02）**，因为：
  - V1 的 **answer = 准确率金标准**（RL 全程不能让答案漂离它）；
  - V1 的 **think = 冷启动改写的原料**（Kimi 把 V1 自己的机器腔 think 改写成自然腔种子）。
- 评测口径：**student = V1 + LoRA（冷启动/RFT/DPO/GRPO 后），teacher = 原始 V1**——看 think 变没变自然、answer 有没有守住原始 V1 的准确率。

**流程**：V1 自产 think+answer → Kimi 改写+打分 V1 的 think → 证伪奖励 → 建冷启动集 → **在 V1 上冷启动 LoRA SFT** → cs-RFT → DPO → GRPO → 评测。

## 1. 硬件（推荐：bf16 LoRA + 2×80GB）

| 资源 | 配置 | 说明 |
|---|---|---|
| **GPU** | **2× 80GB**（A100/H100 80GB）或 2× 96GB | GRPO 峰值 ~115–135GB，单卡放不下；base 用 `device_map="auto"` 分片到 2 卡 |
| 显存账 | base 64GB(bf16) + 激活/KV/训练 ~50–70GB | **GRPO 是瓶颈**；SFT / 冷启动 / DPO 更轻，GRPO 能放下其余都能放下 |
| **内存(RAM)** | **≥256GB 稳**（128GB 勉强） | 加载 64GB 权重 + dataloader + 本地奖励(CPU) |
| **磁盘** | **≥200GB 空闲** | V1 权重 ~64GB + 各阶段 LoRA + rollout 数据 |
| 时间 | 单步约为 14B 的 2–3 倍 | GRPO 100 步会明显更久 |

**预算极紧的退路**：单卡 80/96GB + **4-bit QLoRA**（base 4-bit ≈ 18–20GB，单卡放得下，改动反而最少）。
⚠️ **但量化可能损 V1 已有的准确率，与"不掉准确率"目标直接冲突——若用必须和 bf16 小样本对比确认准确率没塌。**

## 2. config 要改的常量

| 常量 | 14B 值 | 32B 改成 | 原因 |
|---|---|---|---|
| `STUDENT_LOCAL_DIR` | R1-Distill-14B | **V1 32B 路径** | LoRA 的 base = V1 本体 |
| 加载精度 | bf16 | bf16（QLoRA 才开 4bit） | 不伤精度 |
| device_map | "auto"（实际单卡） | "auto"（2 卡分片）/ FSDP | 模型并行 |
| `LORA_TARGET_MODULES` | q/k/v/o(+mlp) | 同，但**先确认 V1 架构的层名** | 适配 32B 结构 |
| `GRPO_PROMPTS_PER_STEP` | 8 | **4–6**（先小） | 控显存 |
| `GRPO_K`（组大小） | 4 | 4（或 3） | 控显存 |
| `GRPO_MAX_FORWARD_LEN` | 3072 | **2048–2560** | 32B 单条前向更贵 |
| `RL_GEN_MAX_NEW_TOKENS` | 当前值 | 视情况下调 | 控生成显存 |
| step04/09 的 batch / grad_acc | 当前值 | **batch 调小、grad_acc 调大** | 等效 batch 不变、省显存 |
| `GRPO_KL_THINK` / `GRPO_KL_ANSWER` | 0.02 / 0.1 | **沿用** | 非对称 KL 机制不变 |
| 采样上限 `RL_ROLLOUT_MAX_QUERIES` / `SEED_MAX` | 400 / 800 | **建议改全量 + 随机轮换** | 32B 要把数据吃满（沙盒已暴露"只覆盖约 2/3"的局限） |

## 3. 必须动的代码（14B 代码是单卡假设）

- **多卡张量落点**：GRPO（`step10_grpo.py`）/ DPO（`step09_dpo.py`）的自定义训练循环里有 `.to(model.device)`。2 卡 `device_map="auto"` 下模型跨设备，**输入要送到第一层所在卡、loss 在最后一层所在卡**。把这些 `.to()` 改成对应设备（输入设备可取 `model.get_input_embeddings().weight.device`）。
- **参考 adapter 多卡**：GRPO 的 `set_adapter("ref")/("default")` + 分片 base 仍可用（PEFT 多 adapter 共享同一份分片 base），但要冒烟确认分片下前向正常。
- **step05 评测推理**：已支持 base+LoRA；32B 加 `device_map="auto"` 即可跨卡推理（参考新写的 `eval_base14b.py` 的 sdpa + 批量思路）。
- **若改用 QLoRA**：在加载处加 `BitsAndBytesConfig(load_in_4bit=True, bnb_4bit_compute_dtype=bfloat16, bnb_4bit_quant_type="nf4")`，单卡即可、**代码改动最少**——但接受精度风险。

## 4. 冒烟（正式烧卡前必做，别省）

1. **加载冒烟**：`device_map="auto"` 加载 V1 32B + 一个空 LoRA，`nvidia-smi` 看两卡分布、能 forward 一条即可。
2. **GRPO 2 步冒烟**：用 `--smoke`（prompts_per_step=2、K=4、steps=2）跑，盯 `nvidia-smi` **峰值**——确认不 OOM、留出余量，再放全量。
3. **奖励先证伪**：在 32B 的自然种子上跑 step13 信号探针，**AUC ≥ 0.7** 才放行大训（先廉价证伪、再烧卡）。

## 5. 运行顺序（命令骨架；精确 stage id 以 `run.py` 为准）

```
# 0) V1 自产 think+answer（= 准确率金标准 + 冷启动原料）
python run.py --only <step02: 调 V1 生成>     # 产出 COMPANY_OUTPUTS

# 1) 冷启动种子：Kimi 改写 V1 的 think + 打分 + 证伪 + 建集
python run.py --only <step11 改写>
python run.py --only <step12 打分>
python run.py --only <step13 证伪>            # AUC≥0.7
python run.py --only 14_build_coldstart

# 2) 在 V1 上冷启动 LoRA SFT → cs-RFT
python run.py --only cs_train                 # base = V1（不是蒸馏学生）
python run.py --from cs_rollout --to cs_rft_report

# 3) DPO → GRPO（先冒烟）
python run.py --from 20_dpo_rollout --to 09f_dpo_report
python run.py --only 10_grpo_train --smoke    # 先 2 步看显存
python run.py --from 10_grpo_train --to 10d_grpo_infer

# 4) 评测：student = V1+GRPO-LoRA  vs  teacher = 原始 V1
python run.py --from <eval infer> --to <eval report>
```

## 6. 风险与红线

- **红线：不掉 V1 准确率。** 三重保险沿用 14B：answer 段重 KL(0.1) 按住 / 奖励准确率门控（答错=0）/ DPO·GRPO 参考锚在上一阶段（不是裸 V1）。
- **先证伪再烧卡**：每次上 GPU 前 step13 AUC≥0.7。
- **数据吃满**：32B 别再死取前 400/800，改**全量 + 随机轮换**采样。
- **量化谨慎**：QLoRA 省钱但可能损 V1 精度；若用，必须 bf16 小样本对比确认准确率没塌。
- **评测同批**：5 个阶段的 student+teacher 尽量放进**同一批 Kimi 重判**，避免批次波动（沙盒里 teacher humanness 0.16↔0.29 的飘）。

---

## 7. 成功判据（迁移算成功的标准）
- V1 的 **think humanness 显著上升**（参考 14B：0.21→0.85 量级），**RAG 显式痕迹（explicit_ref / ref_enumeration）大幅下降**；
- **answer 准确率守住原始 V1 水平**（correct/总数、correct+partial/总数 不显著下降）；
- 全程**纯自我改进**（V1 自己的 think 被 Kimi 洗风格、自己生成自己打分），不引入更强外部老师。
